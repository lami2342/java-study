## 1번 - Java 다형성

### 답: BDCDD

### 풀이
A 클래스와 B 클래스에서 B가 A를 상속받아 메서드를 오버라이딩한다. Java의 다형성(Polymorphism)에 의해, 부모 타입의 참조 변수로 자식 객체를 참조할 때 오버라이딩된 메서드가 호출된다. 이는 동적 바인딩(Dynamic Binding)에 의한 것으로, 컴파일 시점이 아닌 런타임 시점에 실제 객체의 메서드가 호출된다.

- **오버라이딩(Overriding)**: 상위 클래스의 메서드를 하위 클래스에서 재정의
- **다형성**: 하나의 참조 변수가 여러 타입의 객체를 참조할 수 있는 특성

## 2번 - 개방형 표준 프로토콜

### 답: OAuth

### 풀이
**OAuth(Open Authorization)**는 인터넷 사용자들이 비밀번호를 제공하지 않고 다른 웹사이트 상의 자신들의 정보에 대해 접근 권한을 부여할 수 있는 개방형 표준 프로토콜이다.

주요 특징:
- 사용자가 제3자 애플리케이션에 아이디/패스워드를 직접 제공하지 않음
- **액세스 토큰(Access Token)**을 발급하여 권한을 위임
- Google, Facebook, Kakao 등 다양한 플랫폼에서 사용
- OAuth 1.0과 OAuth 2.0 버전이 존재하며, 현재는 OAuth 2.0이 주로 사용됨
- 인증(Authentication)이 아닌 **인가(Authorization)**에 초점을 맞춘 프로토콜

## 3번 - Linux 파일 권한

### 답:
- **(1) chmod**
- **(2) 751**

### 풀이
Linux에서 파일 권한은 **chmod** 명령어로 설정하며, 8진수로 표현한다.

| 구분 | 읽기(r) | 쓰기(w) | 실행(x) | 합계 |
|------|---------|---------|---------|------|
| 사용자(User) | 4 | 2 | 1 | **7** |
| 그룹(Group) | 4 | 0 | 1 | **5** |
| 기타(Other) | 0 | 0 | 1 | **1** |

- 읽기(r) = 4, 쓰기(w) = 2, 실행(x) = 1
- 사용자: r+w+x = 4+2+1 = **7**
- 그룹: r+x = 4+1 = **5**
- 기타: x = 1 = **1**

완성된 명령어: `chmod 751 a.txt`

## 4번 - C언어 완전수

### 답: 34

### 풀이
**완전수(Perfect Number)**란 자기 자신을 제외한 약수의 합이 자기 자신과 같은 수이다.

1부터 1000 사이의 완전수를 찾아보면:

- **6**: 약수 = 1, 2, 3 → 1+2+3 = 6 (완전수)
- **28**: 약수 = 1, 2, 4, 7, 14 → 1+2+4+7+14 = 28 (완전수)
- **496**: 약수 = 1, 2, 4, 8, 16, 31, 62, 124, 248 → 합 = 496 (완전수)

그런데 문제의 복원 답안에 따르면 결과는 **34**이므로, 프로그램의 조건에 따라 6 + 28 = 34가 출력된다. 이는 프로그램에서 탐색 범위나 조건이 제한되어 6과 28만 해당되었을 수 있다.

## 5번 - C언어 구조체 멤버 접근

### 답: -> (화살표 연산자)

### 풀이
C언어에서 구조체 멤버에 접근하는 방법은 두 가지이다:

- **직접 접근**: 구조체 변수를 통해 접근할 때 `.`(점) 연산자 사용
  - 예: `s.name`
- **간접 접근**: 구조체 포인터를 통해 접근할 때 `->`(화살표) 연산자 사용
  - 예: `p->name` (이것은 `(*p).name`과 동일)

```c
struct Student {
    char name[20];
    int age;
};

struct Student s;
struct Student *p = &s;

s.age = 20;      // 직접 접근 (점 연산자)
p->age = 20;     // 간접 접근 (화살표 연산자)
(*p).age = 20;   // 간접 접근 (역참조 후 점 연산자)
```

## 6번 - SQL UNION

### 답: 4, 3, 2, 1

### 풀이
**UNION**은 두 개 이상의 SELECT 문의 결과를 합치는 연산이다.

- **UNION**: 중복을 제거하고 결과를 합침
- **UNION ALL**: 중복을 포함하여 결과를 합침

두 SELECT 문의 결과를 UNION으로 결합하면 중복이 제거된 값들이 합쳐지고, ORDER BY DESC로 내림차순 정렬하면 결과는 **4, 3, 2, 1**이 된다.

## 7번 - 접근 통제

### 답:
- **(1) MAC (Mandatory Access Control, 강제적 접근 통제)**
- **(2) RBAC (Role-Based Access Control, 역할 기반 접근 통제)**
- **(3) DAC (Discretionary Access Control, 임의적 접근 통제)**

### 풀이

| 구분 | MAC | RBAC | DAC |
|------|-----|------|-----|
| 권한 부여 주체 | 시스템(중앙) | 중앙 관리자 | 데이터 소유자 |
| 접근 기준 | 보안 등급(레이블) | 역할(Role) | 신분(Identity) |
| 장점 | 보안성 높음 | 관리 용이 | 유연성 높음 |
| 단점 | 유연성 낮음 | 역할 설계 필요 | 보안성 낮음 |
| 적용 예 | 군사/국방 시스템 | 기업 내부 시스템 | 일반 운영체제 |

- **MAC**: 관리자가 보안 등급을 부여하고 시스템이 강제로 접근을 통제
- **RBAC**: 사용자에게 역할을 할당하고 역할에 따라 권한을 부여
- **DAC**: 객체의 소유자가 자신의 판단에 따라 다른 사용자에게 접근 권한을 부여

## 8번 - C언어 재귀함수

### 답: 5040

### 풀이
팩토리얼(Factorial) 재귀함수로 f(7)을 계산한다.

```
f(7) = 7 × f(6)
     = 7 × 6 × f(5)
     = 7 × 6 × 5 × f(4)
     = 7 × 6 × 5 × 4 × f(3)
     = 7 × 6 × 5 × 4 × 3 × f(2)
     = 7 × 6 × 5 × 4 × 3 × 2 × f(1)
     = 7 × 6 × 5 × 4 × 3 × 2 × 1
     = 5040
```

재귀함수의 핵심:
- **기저 조건(Base Case)**: n이 1일 때 1을 반환
- **재귀 호출**: n × f(n-1)로 자기 자신을 호출
- 7! = 7 × 6 × 5 × 4 × 3 × 2 × 1 = **5040**

## 9번 - 자료 전송 방식

### 답: ATM (Asynchronous Transfer Mode, 비동기 전송 모드)

### 풀이
**ATM(Asynchronous Transfer Mode)**은 고정 길이의 **53바이트 셀(Cell)**을 사용하여 데이터를 전송하는 방식이다.

셀 구조 (총 53바이트):
- **헤더(Header)**: 5바이트 - 경로 정보 포함
- **페이로드(Payload)**: 48바이트 - 실제 데이터

주요 특징:
- 고정 길이 셀을 사용하여 처리 속도가 빠름
- 음성, 영상, 데이터 등 다양한 트래픽을 하나의 네트워크에서 통합 처리 가능
- 가상 회선(Virtual Circuit) 방식 사용
- 연결 지향적(Connection-Oriented) 서비스 제공

## 12번 - 네트워크 기술

### 답: NAT (Network Address Translation, 네트워크 주소 변환)

### 풀이
**NAT(Network Address Translation)**는 사설 IP 주소와 공인 IP 주소 간의 변환을 수행하는 기술이다.

주요 특징:
- **사설 IP → 공인 IP**: 내부 네트워크에서 외부 인터넷으로 나갈 때 변환
- **공인 IP → 사설 IP**: 외부에서 들어오는 패킷을 내부 호스트로 전달할 때 변환
- IPv4 주소 부족 문제를 해결하는 데 기여
- 내부 네트워크 구조를 외부에 숨겨 보안성 향상

NAT의 종류:
- **Static NAT**: 사설 IP와 공인 IP를 1:1로 매핑
- **Dynamic NAT**: 사설 IP를 공인 IP 풀에서 동적으로 할당
- **PAT(Port Address Translation)**: 하나의 공인 IP에 포트 번호를 다르게 하여 여러 사설 IP를 매핑 (NAPT라고도 함)

## 13번 - Java 에러 위치

### 답: 7

### 풀이
Java에서 **static 메서드**(예: main 메서드)는 클래스 레벨에서 동작하므로, 인스턴스 변수에 직접 접근할 수 없다.

```java
// 예시 구조
class Example {
    int x = 10;           // 인스턴스 변수

    public static void main(String[] args) {
        System.out.println(x);  // 에러! static에서 인스턴스 변수 접근 불가
    }
}
```

- **static 멤버**: 클래스가 로딩될 때 메모리에 할당, 객체 생성 없이 사용 가능
- **인스턴스 멤버**: 객체가 생성될 때 메모리에 할당, 객체를 통해서만 접근 가능
- static 메서드에서 인스턴스 변수를 사용하려면 객체를 생성한 후 접근해야 함

해당 코드의 **7번 라인**에서 에러가 발생한다.

## 14번 - Python 입출력

### 답: split

### 풀이
Python의 **split()** 메서드는 문자열을 특정 구분자(delimiter)를 기준으로 분리하여 리스트로 반환한다.

```python
# 기본 사용법 (공백 기준 분리)
text = "Hello World Python"
result = text.split()
print(result)  # ['Hello', 'World', 'Python']

# 특정 구분자 지정
data = "apple,banana,cherry"
result = data.split(",")
print(result)  # ['apple', 'banana', 'cherry']

# input과 함께 사용
a, b = input().split()  # 입력값을 공백 기준으로 분리
```

관련 메서드:
- **split()**: 문자열을 분리하여 리스트로 반환
- **join()**: 리스트를 하나의 문자열로 합침
- **strip()**: 문자열 양쪽의 공백 제거

## 16번 - 소프트웨어 테스트

### 답: 동등분할 (동치분할, Equivalence Partitioning)

### 풀이
**동등분할(Equivalence Partitioning)**은 입력 데이터를 유사한 특성을 가진 그룹(동등 클래스)으로 나누고, 각 그룹에서 대표값을 선택하여 테스트하는 **블랙박스 테스트** 기법이다.

핵심 개념:
- 입력 도메인을 **유효 클래스**와 **무효 클래스**로 분류
- 각 클래스에서 대표값 하나만 테스트해도 같은 클래스의 다른 값과 동일한 결과를 기대
- 테스트 케이스 수를 줄이면서 효과적인 테스트 가능

예시 (나이 입력: 1~120 유효):
| 클래스 | 범위 | 대표값 | 예상 결과 |
|--------|------|--------|-----------|
| 무효 클래스 1 | 0 이하 | -1 | 에러 |
| 유효 클래스 | 1 ~ 120 | 50 | 정상 |
| 무효 클래스 2 | 121 이상 | 150 | 에러 |

관련 테스트 기법:
- **경계값 분석(Boundary Value Analysis)**: 경계값을 중심으로 테스트
- **결정 테이블 테스트**: 조건과 행동의 조합을 테이블로 정리

## 18번 - 라우팅 프로토콜

### 답: RIP (Routing Information Protocol)

### 풀이
**RIP(Routing Information Protocol)**는 **거리 벡터(Distance Vector)** 방식의 라우팅 프로토콜이다.

주요 특징:
- **홉 카운트(Hop Count)**를 메트릭(비용)으로 사용
- 최대 홉 수: **15** (16홉 이상은 도달 불가능으로 간주)
- 주기적으로(보통 30초마다) 라우팅 테이블 전체를 이웃 라우터와 교환
- 소규모 네트워크에 적합
- **벨만-포드(Bellman-Ford)** 알고리즘 기반

버전:
- **RIPv1**: 클래스풀(Classful), 브로드캐스트 사용
- **RIPv2**: 클래스리스(Classless), 멀티캐스트 사용, 인증 기능

비교:
| 프로토콜 | 방식 | 메트릭 | 특징 |
|----------|------|--------|------|
| RIP | 거리 벡터 | 홉 카운트 | 소규모, 최대 15홉 |
| OSPF | 링크 상태 | 대역폭(Cost) | 대규모, 다익스트라 알고리즘 |
| BGP | 경로 벡터 | 정책 기반 | AS 간 라우팅 |

## 19번 - 관계 대수 기호

### 답:
- **(1) 조인(Join): ⋈**
- **(2) 프로젝트(Project): π (파이)**
- **(3) 셀렉트(Select): σ (시그마)**
- **(4) 디비전(Division): ÷**

### 풀이
**관계 대수(Relational Algebra)**는 관계형 데이터베이스에서 원하는 데이터를 얻기 위한 절차적 언어이다.

| 연산 | 기호 | 설명 |
|------|------|------|
| 셀렉트(Select) | **σ** (시그마) | 조건을 만족하는 **행(튜플)**을 선택 |
| 프로젝트(Project) | **π** (파이) | 원하는 **열(속성)**을 선택 |
| 조인(Join) | **⋈** (보타이) | 두 릴레이션을 조건에 따라 결합 |
| 디비전(Division) | **÷** | R÷S: S의 모든 튜플과 관련 있는 R의 튜플 반환 |

기타 관계 대수 연산:
- **합집합(Union)**: ∪
- **차집합(Difference)**: -
- **교집합(Intersection)**: ∩
- **카티션 프로덕트(Cartesian Product)**: ×

## 20번 - 데이터베이스 무결성

### 답: 참조 무결성 (Referential Integrity)

### 풀이
**참조 무결성(Referential Integrity)**은 외래키(Foreign Key)가 참조하는 테이블의 기본키(Primary Key) 값이 존재하거나 NULL이어야 한다는 제약 조건이다.

데이터베이스 무결성의 종류:

| 무결성 | 설명 | 예시 |
|--------|------|------|
| **개체 무결성** (Entity) | 기본키는 NULL이거나 중복될 수 없음 | 학번은 반드시 존재하고 유일해야 함 |
| **참조 무결성** (Referential) | 외래키 값은 참조 테이블의 기본키 값이거나 NULL | 수강 테이블의 학번은 학생 테이블에 존재해야 함 |
| **도메인 무결성** (Domain) | 속성 값은 정의된 도메인(범위) 내의 값이어야 함 | 성별은 'M' 또는 'F'만 가능 |
| **키 무결성** (Key) | 릴레이션에는 최소 하나의 키가 존재해야 함 | 모든 테이블은 기본키를 가져야 함 |
| **NULL 무결성** | NOT NULL 제약이 설정된 속성은 NULL 불가 | 이름 필드는 NULL이 될 수 없음 |

참조 무결성 위반 시 조치 옵션:
- **CASCADE**: 참조하는 행도 함께 삭제/변경
- **SET NULL**: 외래키를 NULL로 설정
- **SET DEFAULT**: 외래키를 기본값으로 설정
- **RESTRICT / NO ACTION**: 삭제/변경을 거부
