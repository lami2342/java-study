## 1장. 포인터 완벽 마스터

> "포인터를 정복하면 C언어의 70%를 정복한 것이다"

### 1.1 포인터 기초

#### 포인터란?
**메모리 주소를 저장하는 변수**이다. 일반 변수가 "값"을 저장한다면, 포인터 변수는 "주소"를 저장한다.

#### 포인터 선언과 초기화

```c
int a = 10;       // 일반 변수: 값 10을 저장
int *p = &a;      // 포인터 변수: a의 주소를 저장

// 메모리 구조 (시각화)
// 변수명:    a          p
// 주소:   [1000]     [2000]
// 값:       10        1000  ← a의 주소!
```

#### 핵심 연산자 2개

| 연산자 | 이름 | 의미 | 예시 |
|---|---|---|---|
| `&` | 주소 연산자 | 변수의 주소를 구한다 | `&a` → a의 주소 |
| `*` | 간접참조 연산자 | 주소가 가리키는 값을 구한다 | `*p` → p가 가리키는 값 |

```c
#include <stdio.h>
int main() {
    int a = 10;
    int *p = &a;

    printf("%d\n", a);    // 10       (a의 값)
    printf("%d\n", &a);   // 1000     (a의 주소, 예시)
    printf("%d\n", p);    // 1000     (p에 저장된 값 = a의 주소)
    printf("%d\n", *p);   // 10       (p가 가리키는 곳의 값)
    printf("%d\n", &p);   // 2000     (p 자체의 주소)
    return 0;
}
```

#### 포인터 연산 (p+1의 의미)

**포인터에 1을 더하면, 해당 자료형 크기만큼 주소가 증가한다!**

```c
int arr[5] = {10, 20, 30, 40, 50};
int *p = arr;  // p는 arr[0]의 주소를 가리킴

// int는 4바이트이므로:
// p     → arr[0]의 주소 (예: 1000)
// p+1   → arr[1]의 주소 (1004, 4바이트 증가!)
// p+2   → arr[2]의 주소 (1008)

printf("%d\n", *p);       // 10  (arr[0])
printf("%d\n", *(p+1));   // 20  (arr[1])
printf("%d\n", *(p+2));   // 30  (arr[2])
```

**암기 공식:**
```
*(p + i)  ==  p[i]  ==  arr[i]
```

#### 포인터와 배열의 관계

```c
int arr[3] = {10, 20, 30};
int *p = arr;  // 배열명 = 첫 번째 원소의 주소

// 모두 같은 의미!
arr[0]   ==  *arr      ==  *p       ==  p[0]
arr[1]   ==  *(arr+1)  ==  *(p+1)   ==  p[1]
arr[2]   ==  *(arr+2)  ==  *(p+2)   ==  p[2]
```

### 1.2 문자열과 포인터

#### char 배열 vs char* 포인터

```c
char str1[] = "KOREA";    // 배열: 수정 가능, 크기 6 (null 포함)
char *str2 = "KOREA";     // 포인터: 문자열 상수를 가리킴

// 메모리 구조
// str1: ['K']['O']['R']['E']['A']['\0']  ← 배열 (스택에 복사)
// str2: ──────→ "KOREA" (문자열 상수 영역)
```

#### 문자열 포인터 접근법 (★★★ 최빈출)

```c
char *p = "KOREA";

// p가 가리키는 메모리:
// 주소:  100  101  102  103  104  105
// 값:    'K'  'O'  'R'  'E'  'A'  '\0'
// p는 100번지를 가리킴

printf("%s\n", p);      // KOREA   (p부터 \0까지 전체 문자열)
printf("%s\n", p+1);    // OREA    (p+1부터 \0까지)
printf("%s\n", p+3);    // EA      (p+3부터 \0까지)

printf("%c\n", *p);     // K       (p가 가리키는 문자 하나)
printf("%c\n", *(p+1)); // O       (p+1이 가리키는 문자 하나)
printf("%c\n", *(p+2)); // R       (p+2가 가리키는 문자 하나)

// 추가 주의사항!
printf("%c\n", *p+1);   // L       (*p는 'K'=75, 75+1=76='L')
printf("%c\n", *p+2);   // M       ('K'=75, 75+2=77='M')
```

> **시험 최대 함정!**
> - `*(p+2)` → p를 2칸 이동 후 값 참조 → `'R'`
> - `*p+2` → p가 가리키는 값에 2를 더함 → `'K'+2` = `'M'`
> - 괄호 위치가 결과를 완전히 바꾼다!

#### printf에서 %s와 %c의 차이

```c
char *p = "HELLO";

printf("%s", p);     // HELLO  ← 주소부터 \0까지 전체 출력
printf("%c", *p);    // H      ← 한 문자만 출력
printf("%s", p+2);   // LLO    ← p+2 주소부터 \0까지 출력
printf("%c", *(p+2));// L      ← p+2 위치의 한 문자만 출력
```

#### 실전 예제: 문자열 포인터 종합

```c
#include <stdio.h>
int main() {
    char *p = "COMPUTER";

    printf("%c\n", *(p+3));    // P    (4번째 문자)
    printf("%c\n", *p+3);     // F    ('C'=67, 67+3=70='F')
    printf("%s\n", p+5);      // TER  (6번째 문자부터 끝까지)
    printf("%c\n", *(p+7));   // R    (8번째 문자)

    p = p + 3;                 // p가 이제 'P'를 가리킴
    printf("%s\n", p);         // PUTER
    printf("%c\n", *p);       // P
    printf("%c\n", *(p+2));   // T

    return 0;
}
```

### 1.3 배열 포인터 vs 포인터 배열

> **이것은 시험에서 반드시 구분할 수 있어야 한다!**

#### 핵심 비교표

| 구분 | 배열 포인터 | 포인터 배열 |
|---|---|---|
| 선언 | `int (*p)[3]` | `int *p[3]` |
| 의미 | 크기 3인 배열을 가리키는 포인터 1개 | 포인터 3개를 담는 배열 |
| 읽는법 | "3개짜리 int 배열의 포인터" | "int 포인터 3개의 배열" |
| 용도 | 2차원 배열 접근 | 여러 변수/배열을 가리킴 |

#### 배열 포인터 `int (*p)[3]`

```c
int arr[2][3] = {{1, 2, 3}, {4, 5, 6}};
int (*p)[3] = arr;  // 2차원 배열을 행 단위로 가리킴

// p     → arr[0] = {1, 2, 3}
// p+1   → arr[1] = {4, 5, 6}

printf("%d\n", (*p)[0]);     // 1   (0행 0열)
printf("%d\n", (*p)[2]);     // 3   (0행 2열)
printf("%d\n", (*(p+1))[0]); // 4   (1행 0열)
printf("%d\n", (*(p+1))[2]); // 6   (1행 2열)

// 또는 이렇게도 접근 가능
printf("%d\n", p[0][1]);     // 2
printf("%d\n", p[1][2]);     // 6
```

#### 포인터 배열 `int *p[3]`

```c
int a = 10, b = 20, c = 30;
int *p[3];       // int 포인터 3개짜리 배열

p[0] = &a;
p[1] = &b;
p[2] = &c;

printf("%d\n", *p[0]);   // 10
printf("%d\n", *p[1]);   // 20
printf("%d\n", *p[2]);   // 30
```

#### 실전 예제 (기출 변형)

```c
#include <stdio.h>
int main() {
    int arr[2][3] = {{1, 2, 3}, {4, 5, 6}};
    int (*p)[3] = arr;

    printf("%d\n", **p);          // 1  (**p = *(*(p+0)+0) = arr[0][0])
    printf("%d\n", **(p+1));      // 4  (*(*(p+1)+0) = arr[1][0])
    printf("%d\n", *(*p+1));      // 2  (*(*(p+0)+1) = arr[0][1])
    printf("%d\n", *(*(p+1)+2));  // 6  (arr[1][2])

    return 0;
}
```

**2차원 배열 포인터 접근 공식:**
```
*(*(p + i) + j)  ==  p[i][j]  ==  arr[i][j]
```

### 1.4 이중 포인터

#### 이중 포인터란?

**포인터의 주소를 저장하는 포인터**이다.

```c
int a = 10;
int *p = &a;      // p는 a의 주소를 저장
int **pp = &p;    // pp는 p의 주소를 저장

// 메모리 구조:
//  변수:    a         p          pp
//  주소: [1000]    [2000]     [3000]
//  값:     10       1000       2000
//
//  pp → p → a
//  **pp = *(*pp) = *(p) = a = 10
```

```c
printf("%d\n", a);      // 10
printf("%d\n", *p);     // 10
printf("%d\n", **pp);   // 10   (모두 같은 값!)

printf("%d\n", &a);     // 1000
printf("%d\n", p);      // 1000
printf("%d\n", *pp);    // 1000 (모두 a의 주소!)
```

#### 이중 포인터와 배열 (★★★ 빈출)

```c
#include <stdio.h>
int main() {
    int arr[3] = {10, 20, 30};
    int *p = arr;
    int **pp = &p;

    printf("%d\n", **pp);       // 10  (*p = arr[0])
    printf("%d\n", *(*pp+1));   // 20  (*(p+1) = arr[1])
    printf("%d\n", *(*pp+2));   // 30  (*(p+2) = arr[2])

    (*pp)++;   // p = p + 1 (p가 arr[1]을 가리킴)
    printf("%d\n", **pp);       // 20

    return 0;
}
```

#### 이중 포인터 해석 비법

```
**pp 해석 순서:
1단계: pp가 가리키는 곳 → *pp (= p)
2단계: p가 가리키는 곳 → **pp (= *p = a)

항상 안에서 밖으로, *를 하나씩 벗겨내기!
```

### 1.5 구조체 포인터

#### 기본 문법

```c
struct Student {
    char name[20];
    int score;
};

struct Student s1 = {"Kim", 90};
struct Student *p = &s1;

// 멤버 접근 방법 2가지
printf("%d\n", s1.score);     // 90  (직접 접근: 변수.멤버)
printf("%d\n", (*p).score);   // 90  (포인터 역참조 후 접근)
printf("%d\n", p->score);     // 90  (화살표 연산자: 포인터->멤버)
```

> **시험에서는 `->` 연산자가 가장 많이 출제된다!**

#### 핵심 공식

```
p->member  ==  (*p).member
```

- 구조체 변수: `.` (점) 사용
- 구조체 포인터: `->` (화살표) 사용

#### 구조체 배열의 포인터

```c
struct Student arr[3] = {
    {"Kim", 90},
    {"Lee", 85},
    {"Park", 95}
};

struct Student *p = arr;  // 배열의 첫 번째 원소를 가리킴

printf("%s\n", p->name);       // Kim
printf("%d\n", (p+1)->score);  // 85  (arr[1].score)
printf("%s\n", (p+2)->name);   // Park
printf("%d\n", p[1].score);    // 85  (배열 인덱스 접근도 가능)
```

### 🔥 포인터 킬러 문제 패턴 5선

#### 패턴 1: 문자열 포인터 + 연산 혼합

```c
#include <stdio.h>
int main() {
    char *p = "ABCDE";
    printf("%c", *(p+2));    // ?
    printf("%c", *p+2);     // ?
    printf("%s", p+3);      // ?
    return 0;
}
```
**정답:** `C` `C` `DE`
- `*(p+2)`: p를 2칸 이동 → 'C'
- `*p+2`: 'A'(65) + 2 = 67 = 'C'
- `p+3`: 3번째부터 문자열 출력 → "DE"

#### 패턴 2: 배열 포인터 + 2차원 배열

```c
#include <stdio.h>
int main() {
    int a[2][3] = {{1, 2, 3}, {4, 5, 6}};
    int (*p)[3] = a;

    printf("%d\n", *(*(p+1)+1));  // ?
    printf("%d\n", **p);          // ?
    return 0;
}
```
**정답:** `5` `1`
- `*(*(p+1)+1)` = a[1][1] = 5
- `**p` = a[0][0] = 1

#### 패턴 3: 포인터 배열 + 문자열

```c
#include <stdio.h>
int main() {
    char *arr[3] = {"apple", "banana", "cherry"};
    char **pp = arr;

    printf("%s\n", *pp);         // ?
    printf("%s\n", *(pp+1));     // ?
    printf("%c\n", **(pp+2));    // ?
    printf("%c\n", *(*(pp+1)+2)); // ?
    return 0;
}
```
**정답:**
- `*pp` → "apple"
- `*(pp+1)` → "banana"
- `**(pp+2)` → 'c' (cherry의 첫 글자)
- `*(*(pp+1)+2)` → 'n' (banana의 3번째 글자)

#### 패턴 4: 포인터 매개변수 (값 변경)

```c
#include <stdio.h>
void func(int *a, int *b) {
    *a = *a + *b;
    *b = *a - *b;
    *a = *a - *b;
}
int main() {
    int x = 10, y = 20;
    func(&x, &y);
    printf("%d %d\n", x, y);  // ?
    return 0;
}
```
**정답:** `20 10`
- 이 함수는 두 값을 교환(swap)하는 함수!
- 추적: `*a=10+20=30`, `*b=30-20=10`, `*a=30-10=20`

#### 패턴 5: 포인터 증감 연산

```c
#include <stdio.h>
int main() {
    int arr[] = {10, 20, 30, 40, 50};
    int *p = arr;

    printf("%d\n", *p++);    // ?  (후위 증가)
    printf("%d\n", *p);      // ?
    printf("%d\n", *++p);    // ?  (전위 증가)
    printf("%d\n", ++*p);    // ?  (값 증가)
    return 0;
}
```
**정답:**
- `*p++`: 현재 값 출력(10) 후 p 이동 → `10`
- `*p`: p는 arr[1] → `20`
- `*++p`: p를 먼저 이동(arr[2]) 후 값 출력 → `30`
- `++*p`: 현재 값(30)에 1 더함 → `31`

> **후위/전위 연산자 우선순위 암기:**
> - `*p++` → `*(p++)` : 값 먼저, 포인터 이동
> - `*++p` → `*(++p)` : 포인터 이동 먼저, 값
> - `++*p` → `++(*p)` : 값을 1 증가
> - `(*p)++` → 값 반환 후 1 증가

### 2.3 구조체 배열

```c
typedef struct {
```

### 3.2 Head 삽입 (가장 빈출!)

> **새 노드를 리스트의 맨 앞에 삽입하는 방식**
> **결과적으로 삽입 순서와 역순으로 저장된다!**

```c
Node* insertAtHead(Node* head, int value) {
    Node* new_node = (Node*)malloc(sizeof(Node));
    new_node->data = value;
    new_node->next = head;   // 새 노드의 next를 기존 head로
    return new_node;         // 새 노드가 새로운 head
}
```

**과정 시각화:**

```
[초기 상태]  head → NULL

[10 삽입]    head → [10|NULL]

[20 삽입]    head → [20|──→] → [10|NULL]
             (20의 next가 기존 head인 10을 가리킴)

[30 삽입]    head → [30|──→] → [20|──→] → [10|NULL]

∴ 삽입 순서: 10, 20, 30
∴ 리스트 순서: 30 → 20 → 10  (역순!)
```

```c
}
```

### 3.4 노드 재연결 (reconnect)

> **포인터만 변경하여 노드 순서를 바꾸는 문제가 자주 출제된다**

#### 특정 노드 삭제

```c
Node* deleteNode(Node* head, int target) {
    // head 노드를 삭제하는 경우
    if (head->data == target) {
        Node* temp = head;
        head = head->next;
        free(temp);
        return head;
    }

    // 중간/끝 노드를 삭제하는 경우
    Node* curr = head;
    while (curr->next != NULL) {
        if (curr->next->data == target) {
            Node* temp = curr->next;
            curr->next = temp->next;  // 핵심: 건너뛰기!
            free(temp);
            return head;
        }
        curr = curr->next;
    }
    return head;
}
```

**삭제 과정 시각화 (20 삭제):**
```
삭제 전:  [10|──→] → [20|──→] → [30|NULL]
                       ↑삭제!
삭제 후:  [10|──→] ─────────→ [30|NULL]
          (10의 next를 30으로 변경)
```

#### 노드 역순 (reverse)

```c
Node* reverse(Node* head) {
    Node* prev = NULL;
    Node* curr = head;
    Node* next = NULL;

    while (curr != NULL) {
        next = curr->next;    // 다음 노드 저장
        curr->next = prev;    // 현재 노드의 방향 반전
        prev = curr;          // prev 이동
        curr = next;          // curr 이동
    }
    return prev;  // 새로운 head
}
```

**역순 과정 추적:**
```
초기:  NULL  ←  [10] → [20] → [30] → NULL
              prev  curr  next

1단계: NULL ← [10]   [20] → [30] → NULL
              prev   curr   next

2단계: NULL ← [10] ← [20]   [30] → NULL
                     prev   curr   next

3단계: NULL ← [10] ← [20] ← [30]   NULL
                            prev   curr

최종:  [30] → [20] → [10] → NULL
       head(=prev)
```

## 4장. 비트 연산 & 16진수

### 4.1 비트 연산자 총정리

| 연산자 | 이름 | 설명 | 예시 (5 op 3) |
|---|---|---|---|
| `&` | AND | 둘 다 1이면 1 | `5 & 3` = `0101 & 0011` = `0001` = **1** |
| `\|` | OR | 하나라도 1이면 1 | `5 \| 3` = `0101 \| 0011` = `0111` = **7** |
| `^` | XOR | 서로 다르면 1 | `5 ^ 3` = `0101 ^ 0011` = `0110` = **6** |
| `~` | NOT | 비트 반전 | `~5` = `~00000101` = `11111010` = **-6** |
| `<<` | 좌 시프트 | 왼쪽으로 이동 | `5 << 1` = `1010` = **10** |
| `>>` | 우 시프트 | 오른쪽으로 이동 | `5 >> 1` = `0010` = **2** |

#### AND 연산 (&) - 마스킹에 사용

```
    0101  (5)
 &  0011  (3)
 --------
    0001  (1)

규칙: 둘 다 1일 때만 1
```

#### OR 연산 (|) - 비트 설정에 사용

```
    0101  (5)
 |  0011  (3)
 --------
    0111  (7)

규칙: 하나라도 1이면 1
```

#### XOR 연산 (^) - 비트 토글에 사용

```
    0101  (5)
 ^  0011  (3)
 --------
    0110  (6)

규칙: 서로 다르면 1, 같으면 0
```

#### NOT 연산 (~) - 비트 반전

```
~5 = -(5+1) = -6

(2의 보수 체계에서 ~n = -(n+1))
```

> **시험 꿀팁: `~n = -(n+1)` 공식 하나만 외우면 NOT 연산 끝!**

### 4.2 16진수 <-> 2진수 변환

#### 변환표 (필수 암기!)

| 16진수 | 2진수 | 10진수 |
|---|---|---|
| 0 | 0000 | 0 |
| 1 | 0001 | 1 |
| 2 | 0010 | 2 |
| 3 | 0011 | 3 |
| 4 | 0100 | 4 |
| 5 | 0101 | 5 |
| 6 | 0110 | 6 |
| 7 | 0111 | 7 |
| 8 | 1000 | 8 |
| 9 | 1001 | 9 |
| A | 1010 | 10 |
| B | 1011 | 11 |
| C | 1100 | 12 |
| D | 1101 | 13 |

### 4.4 시프트 연산 (<< >>)

#### 왼쪽 시프트 (<<)

```
n << k  =  n * 2^k

5 << 1  =  5 * 2  = 10
5 << 2  =  5 * 4  = 20
5 << 3  =  5 * 8  = 40
3 << 4  =  3 * 16 = 48
```

```
5 = 00000101
5 << 1:
  00001010 = 10  (왼쪽으로 1칸, 오른쪽 0 채움)
5 << 2:
  00010100 = 20  (왼쪽으로 2칸)
```

#### 오른쪽 시프트 (>>)

```
n >> k  =  n / 2^k  (소수점 버림)

20 >> 1  =  20 / 2  = 10
20 >> 2  =  20 / 4  = 5
20 >> 3  =  20 / 8  = 2  (2.5에서 소수점 버림)
7 >> 1   =  7 / 2   = 3  (3.5에서 소수점 버림)
```

### 🔥 비트 연산 속산법

```
[속산 공식 정리]

& (AND): 둘 다 1 → 1         (마스킹)
| (OR):  하나만 1 → 1        (설정)
^ (XOR): 다르면 1 → 1        (토글)
~ (NOT): ~n = -(n+1)          (반전)
<< n:    * 2^n                (곱하기)
>> n:    / 2^n (소수 버림)     (나누기)
```

**자주 나오는 비트 연산 결과:**
```
x & 0  = 0        (어떤 수든 0과 AND하면 0)
x | 0  = x        (어떤 수든 0과 OR하면 자기 자신)
x ^ 0  = x        (어떤 수든 0과 XOR하면 자기 자신)
x ^ x  = 0        (같은 수끼리 XOR하면 0!)
x & x  = x        (같은 수끼리 AND하면 자기 자신)
```

### 5.4 문자 변환 공식: (ch - 'A' + n) % 26 + 'A'

> **시저 암호(Caesar Cipher) 패턴 - 자주 출제!**

```c
// 알파벳을 n칸 시프트하는 공식
char shift(char ch, int n) {
    if (isupper(ch))
        return (ch - 'A' + n) % 26 + 'A';
    if (islower(ch))
        return (ch - 'a' + n) % 26 + 'a';
    return ch;  // 알파벳이 아니면 그대로
}

// 예시
shift('A', 3);  // (0 + 3) % 26 + 65 = 68 = 'D'
shift('X', 3);  // (23 + 3) % 26 + 65 = 0 + 65 = 'A' (순환!)
shift('Z', 1);  // (25 + 1) % 26 + 65 = 0 + 65 = 'A' (순환!)
```

**공식 해석:**
```
ch - 'A'   : 알파벳 순서 (A=0, B=1, ..., Z=25)
+ n        : n칸 이동
% 26       : 26을 넘으면 다시 처음으로 (순환)
+ 'A'      : 다시 ASCII로 변환
```

## 6장. 제어문 함정

### 6.1 switch-case의 fall-through (★★★ 최빈출 함정!)

> **break가 없으면 다음 case로 계속 실행된다!**
> **이것이 시험에서 가장 흔한 함정!**

#### 정상적인 switch-case

```c
int n = 2;
switch (n) {
    case 1: printf("ONE\n");   break;
    case 2: printf("TWO\n");   break;    // 여기만 실행
    case 3: printf("THREE\n"); break;
    default: printf("OTHER\n");
}
// 출력: TWO
```

#### break 없는 fall-through (함정!)

```c
int n = 2;
switch (n) {
    case 1: printf("ONE\n");              // break 없음!
    case 2: printf("TWO\n");              // 여기서 시작
    case 3: printf("THREE\n");            // fall-through!
    default: printf("OTHER\n");           // fall-through!
}
// 출력:
// TWO
// THREE
// OTHER
```

> **break가 없으면 매칭된 case부터 아래로 모두 실행!**

#### 기출 변형 문제

```c
#include <stdio.h>
int main() {
    int n = 3;
    int result = 0;

    switch (n) {
        case 1: result += 1;
        case 2: result += 2;
        case 3: result += 3;      // 여기서 시작
        case 4: result += 4;      // fall-through
        default: result += 5;     // fall-through
    }
    printf("%d\n", result);  // ?
    return 0;
}
```
**정답:** `12` (3 + 4 + 5 = 12)

#### 또 다른 변형

```c
#include <stdio.h>
int main() {
    char grade = 'B';

    switch (grade) {
        case 'A':
            printf("Excellent ");
        case 'B':
            printf("Good ");      // 여기서 시작
        case 'C':
            printf("Average ");   // fall-through
            break;                // 여기서 탈출!
        case 'D':
            printf("Poor ");
        default:
            printf("Fail ");
    }
    return 0;
}
```
**정답:** `Good Average` (C에서 break를 만나 탈출)

### 6.2 삼항연산자 중첩

```c
#include <stdio.h>
```

## 7장. 함수와 변수 스코프

### 7.1 static 지역변수 (★★★ 초빈출!)

> **static 지역변수는 함수가 끝나도 값이 유지된다!**
> **이것은 거의 매회 출제된다!**

#### 일반 지역변수 vs static 지역변수

```c
#include <stdio.h>

void func_normal() {
    int count = 0;        // 매번 0으로 초기화!
    count++;
    printf("normal: %d\n", count);
}

void func_static() {
    static int count = 0;  // 최초 1회만 초기화! 이후 값 유지!
    count++;
    printf("static: %d\n", count);
}

int main() {
    func_normal();  // normal: 1
    func_normal();  // normal: 1  (매번 0에서 시작)
    func_normal();  // normal: 1

    func_static();  // static: 1
    func_static();  // static: 2  (이전 값 유지!)
    func_static();  // static: 3
    return 0;
}
```

#### 핵심 정리

| 구분 | 일반 지역변수 | static 지역변수 |
|---|---|---|
| 초기화 | 함수 호출될 때마다 | 최초 1회만 |
| 수명 | 함수 종료 시 소멸 | 프로그램 종료까지 유지 |
| 범위 | 함수 내부만 | 함수 내부만 (변수명 접근 범위는 동일) |
| 초기값 | 쓰레기값 | 0 (자동 초기화) |

#### 기출 필수 예제

```c
#include <stdio.h>

int func(int n) {
    static int sum = 0;
    sum += n;
    return sum;
}

int main() {
    printf("%d\n", func(3));   // ?
    printf("%d\n", func(5));   // ?
    printf("%d\n", func(7));   // ?
    return 0;
}
```
**추적:**
```
func(3): sum = 0 + 3 = 3  → 출력: 3
func(5): sum = 3 + 5 = 8  → 출력: 8  (이전 값 유지!)
func(7): sum = 8 + 7 = 15 → 출력: 15
```
**정답:** `3` `8` `15`

```c
#include <stdio.h>

int g = 10;          // 전역 변수

}
```

### 7.3 재귀 함수 풀이법

> **재귀 = 자기 자신을 호출하는 함수**
> **풀이법: 호출 스택을 그려라!**

#### 팩토리얼 (가장 기본)

```c
int factorial(int n) {
    if (n <= 1) return 1;        // 종료 조건
    return n * factorial(n - 1);  // 재귀 호출
}

printf("%d\n", factorial(5));  // ?
```

**호출 스택 추적:**
```
factorial(5)
= 5 * factorial(4)
= 5 * 4 * factorial(3)
= 5 * 4 * 3 * factorial(2)
= 5 * 4 * 3 * 2 * factorial(1)
= 5 * 4 * 3 * 2 * 1
= 120
```
**정답:** `120`

#### 피보나치 (빈출)

```c
int fib(int n) {
    if (n <= 1) return n;
    return fib(n-1) + fib(n-2);
}

printf("%d\n", fib(6));  // ?
```

**추적 (트리 형태):**
```
fib(6)
= fib(5) + fib(4)
= (fib(4)+fib(3)) + (fib(3)+fib(2))
= ((fib(3)+fib(2))+(fib(2)+fib(1))) + ((fib(2)+fib(1))+fib(2))
...
fib(0)=0, fib(1)=1, fib(2)=1, fib(3)=2, fib(4)=3, fib(5)=5, fib(6)=8
```
**정답:** `8`

#### 재귀로 숫자 출력 (출제 빈도 높음!)

```c
void func(int n) {
    if (n <= 0) return;
    func(n - 1);         // 먼저 재귀
    printf("%d ", n);    // 돌아오면서 출력
}

func(5);  // 출력: 1 2 3 4 5
```

```c
void func2(int n) {
    if (n <= 0) return;
    printf("%d ", n);    // 먼저 출력
    func2(n - 1);        // 후에 재귀
}

func2(5);  // 출력: 5 4 3 2 1
```

> **재귀 풀이 비법:**
> 1. printf가 재귀 호출 **전**에 있으면 → 큰 수부터 (역순)
> 2. printf가 재귀 호출 **후**에 있으면 → 작은 수부터 (정순)

#### 재귀 + 나머지 연산 (자릿수 분리)

```c
void printDigits(int n) {
```

```
  → printDigits(123)

        ← printf(1 % 10) → 1

    ← printf(123 % 10) → 3

```

### 7.4 포인터 매개변수 (Call by Reference)

#### Call by Value vs Call by Reference

```c
#include <stdio.h>

// Call by Value: 복사본 전달 → 원본 변경 불가
void swapByValue(int a, int b) {
    int temp = a;
    a = b;
    b = temp;
    // a, b는 복사본이므로 원본에 영향 없음!
}

// Call by Reference (주소 전달): 원본 변경 가능!
void swapByRef(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
    // *a, *b를 통해 원본 직접 변경!
}

int main() {
    int x = 10, y = 20;

    swapByValue(x, y);
    printf("%d %d\n", x, y);   // 10 20 (변경 안됨!)

    swapByRef(&x, &y);
    printf("%d %d\n", x, y);   // 20 10 (변경됨!)

    return 0;
}
```

#### 배열은 항상 주소 전달!

```c
void modifyArray(int arr[], int size) {
    for (int i = 0; i < size; i++)
        arr[i] *= 2;  // 원본이 변경됨!
}

int main() {
    int arr[] = {1, 2, 3, 4, 5};
    modifyArray(arr, 5);
    // arr = {2, 4, 6, 8, 10}  (원본 변경!)
    return 0;
}
```

> **배열명 자체가 주소이므로, 배열은 항상 원본이 전달된다!**

## 8장. 자료구조 구현

### 8.1 스택 (push/pop/isEmpty)

> **스택 = LIFO (Last In First Out, 후입선출)**
> **접시 쌓기와 같다: 마지막에 넣은 것을 먼저 꺼낸다**

#### 배열로 스택 구현

```c
#include <stdio.h>
#define MAX 100

int stack[MAX];
int top = -1;       // 스택이 비어있는 상태

// 스택이 비었는지 확인
int isEmpty() {
    return top == -1;    // top이 -1이면 비어있음
}

// 스택이 가득 찼는지 확인
int isFull() {
    return top == MAX - 1;
}

// 데이터 삽입
void push(int data) {
    if (isFull()) {
        printf("Stack Overflow!\n");
        return;
    }
    stack[++top] = data;   // top을 먼저 증가시킨 후 저장
}

// 데이터 삭제 및 반환
int pop() {
    if (isEmpty()) {
        printf("Stack Underflow!\n");
        return -1;
    }
    return stack[top--];   // 현재 top 값을 반환 후 감소
}

// 최상위 값 확인 (제거하지 않음)
int peek() {
    if (isEmpty()) return -1;
    return stack[top];
}
```

#### 스택 동작 추적

```c
int main() {
    push(10);    // stack: [10],          top=0
    push(20);    // stack: [10, 20],      top=1
    push(30);    // stack: [10, 20, 30],  top=2

    printf("%d\n", pop());    // 30 (LIFO!)  top=1
    printf("%d\n", pop());    // 20           top=0

    push(40);    // stack: [10, 40],      top=1

    printf("%d\n", pop());    // 40           top=0
    printf("%d\n", pop());    // 10           top=-1

    return 0;
}
```

**스택 시각화:**
```
push(10) push(20) push(30)    pop()      pop()

  [ ]     [  ]    [30] ←top   [  ]       [  ]
  [ ]     [20]    [20]        [20] ←top   [  ]
  [10]    [10]    [10]        [10]        [10] ←top
```

### 8.2 큐 (enqueue/dequeue)

> **큐 = FIFO (First In First Out, 선입선출)**
> **줄서기와 같다: 먼저 온 사람이 먼저 나간다**

#### 배열로 큐 구현 (선형 큐)

```c
#include <stdio.h>
#define MAX 100

int queue[MAX];
int front = 0;     // 꺼낼 위치
int rear = 0;      // 넣을 위치

int isEmpty() {
    return front == rear;
}

void enqueue(int data) {
    queue[rear++] = data;    // rear 위치에 저장 후 증가
}

int dequeue() {
    if (isEmpty()) {
        printf("Queue Empty!\n");
        return -1;
    }
    return queue[front++];   // front 위치 값 반환 후 증가
}
```

#### 큐 동작 추적

```c
int main() {
    enqueue(10);   // queue: [10, _, _]      front=0, rear=1
    enqueue(20);   // queue: [10, 20, _]     front=0, rear=2
    enqueue(30);   // queue: [10, 20, 30]    front=0, rear=3

    printf("%d\n", dequeue());  // 10 (FIFO!) front=1, rear=3
    printf("%d\n", dequeue());  // 20         front=2, rear=3

    enqueue(40);   // queue: [_, _, 30, 40]  front=2, rear=4

    printf("%d\n", dequeue());  // 30         front=3, rear=4
    printf("%d\n", dequeue());  // 40         front=4, rear=4

    return 0;
}
```

**큐 시각화:**
```
enqueue(10) enqueue(20) enqueue(30) dequeue() dequeue()
                                    →10       →20
front→[10]  front→[10]  front→[10]  [  ]      [  ]
      [  ]        [20]        [20]  front→[20] [  ]
      [  ]        [  ]  rear→[30]        [30]  front→[30]
      rear        rear                   rear        rear
```

#### 원형 큐 (Circular Queue) - 심화

```c
#define MAX 5

int queue[MAX];
int front = 0;
int rear = 0;

int isEmpty() {
    return front == rear;
}

int isFull() {
    return (rear + 1) % MAX == front;  // 하나를 비워둠
}

void enqueue(int data) {
    if (isFull()) return;
    queue[rear] = data;
    rear = (rear + 1) % MAX;   // 원형으로 순환!
}

int dequeue() {
    if (isEmpty()) return -1;
    int data = queue[front];
    front = (front + 1) % MAX;  // 원형으로 순환!
    return data;
}
```

### 문제 1. 포인터 + 문자열 (난이도: ★★★)

다음 프로그램의 실행 결과를 쓰시오.

**출력:** `GSAM`

추적:
```
p → P R O G R A M \0
    0 1 2 3 4 5 6
*(p+3) = 'G'
*p + 3 = 'P' + 3 = 80 + 3 = 83 = 'S'
p+5 → "AM"
```

</details>

```
func(0): s = 10 + 0 = 10 → return 10

4 + 3 + 2 + 1 + 0 = 10
```

### 문제 3. 연결리스트 (난이도: ★★★★)

### 문제 9. 스택 구현 (난이도: ★★★)

```c
#include <stdio.h>

int stack[MAX];

void push(int data) { stack[++top] = data; }
int pop() { return stack[top--]; }
int isEmpty() { return top == -1; }

int main() {
    int arr[] = {5, 3, 8, 1, 4};

    for (int i = 0; i < 5; i++) {
        if (arr[i] % 2 == 1)
            push(arr[i]);
    }

    while (!isEmpty())
        printf("%d ", pop());
    return 0;
}
```

```
arr[0]=5 (홀수) → push(5)   stack: [5]
arr[1]=3 (홀수) → push(3)   stack: [5, 3]
arr[2]=8 (짝수) → 건너뜀
arr[3]=1 (홀수) → push(1)   stack: [5, 3, 1]
arr[4]=4 (짝수) → 건너뜀
```

### 문제 10. 종합 문제 (난이도: ★★★★★)

다음 프로그램의 실행 결과를 쓰시오.

```c
#include <stdio.h>

void func(int *arr, int size) {
    int temp;
    for (int i = 0; i < size - 1; i++) {
        for (int j = 0; j < size - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}

int main() {
    int arr[] = {64, 25, 12, 22, 11};
    int size = 5;

    func(arr, size);

    printf("%d %d %d\n", arr[0], arr[2], arr[4]);
    return 0;
}
```

<details>
<summary>정답 확인</summary>

**정답:** `11 22 64`

추적:
```
이 함수는 버블 정렬(Bubble Sort)!
큰 수를 뒤로 보내는 오름차순 정렬

초기:    [64, 25, 12, 22, 11]
정렬 후: [11, 12, 22, 25, 64]

arr[0]=11, arr[2]=22, arr[4]=64
출력: 11 22 64
```

정렬 과정:
```
[64, 25, 12, 22, 11]
→ [25, 12, 22, 11, 64]  (1회차: 64가 맨 뒤로)
→ [12, 22, 11, 25, 64]  (2회차: 25가 뒤에서 2번째로)
→ [12, 11, 22, 25, 64]  (3회차: 22가 중간으로)
→ [11, 12, 22, 25, 64]  (4회차: 완료)
```

</details>

## 부록. 최종 암기 체크리스트

시험 직전에 빠르게 복습할 내용:

### 포인터
- [ ] `*(p+i)` = `p[i]` = `arr[i]`
- [ ] `*(p+i)`와 `*p+i`의 차이 (괄호!)
- [ ] `int (*p)[3]` (배열포인터) vs `int *p[3]` (포인터배열)
- [ ] `**pp` = `*(*pp)` = 이중 포인터 해석
- [ ] `p->member` = `(*p).member`

### 문자열/ASCII
- [ ] 'A'=65, 'a'=97, '0'=48, 대소문자 차이=32
- [ ] `%s`는 주소부터 \0까지, `%c`는 한 문자
- [ ] 시저 암호: `(ch-'A'+n)%26+'A'`

### 제어문
- [ ] switch에서 break 없으면 fall-through!
- [ ] `++i`(전위) vs `i++`(후위) 차이
- [ ] do-while은 최소 1번 실행

### 함수/변수
- [ ] static 변수는 값이 유지된다!
- [ ] 재귀: printf 위치에 따라 출력 순서 달라짐
- [ ] 배열은 함수에 주소로 전달 (원본 변경!)

### 비트연산
- [ ] `&`(AND), `|`(OR), `^`(XOR), `~`(NOT=-n-1)
- [ ] `<<n` = x2^n, `>>n` = /2^n
- [ ] 16진수 한 자리 = 2진수 4자리

### 자료구조
- [ ] Head 삽입 → 삽입 순서의 역순 저장
- [ ] 스택 = LIFO (후입선출)
- [ ] 큐 = FIFO (선입선출)
