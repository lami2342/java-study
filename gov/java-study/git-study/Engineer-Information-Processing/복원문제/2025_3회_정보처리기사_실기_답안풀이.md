## 1번 - UML 다이어그램

### 답: 패키지 (Package)

### 풀이
- **패키지 다이어그램(Package Diagram)**은 UML의 구조 다이어그램 중 하나이다.
- 시스템의 구성 요소를 **폴더 모양의 패키지 단위**로 그룹화하여 표현한다.
- 패키지 간의 의존 관계(dependency)를 점선 화살표로 나타낸다.
- 대규모 시스템을 논리적으로 분류하여 전체 구조를 파악하기 쉽게 해준다.
- UML 구조 다이어그램 종류: 클래스, 객체, 컴포넌트, 배치, 복합구조, **패키지** 다이어그램

## 2번 - 소프트웨어 테스트 커버리지

### 답: ㅇ (조건)

### 풀이
- **조건 커버리지(Condition Coverage)**: 각 개별 조건식이 True와 False를 **각각 한 번 이상** 만족하도록 테스트 케이스를 설계하는 기준이다.
- 주요 커버리지 기준 비교:
  - **문장 커버리지**: 모든 실행 문장을 최소 1회 수행
  - **결정(분기) 커버리지**: 전체 결정문(if, while 등)의 결과가 True/False를 각각 1회 이상 수행
  - **조건 커버리지**: 결정문 내부의 **개별 조건식**이 True/False를 각각 1회 이상 수행
  - **조건/결정 커버리지**: 조건 커버리지 + 결정 커버리지를 동시에 만족
  - **변경 조건/결정 커버리지(MC/DC)**: 각 개별 조건이 전체 결정에 독립적으로 영향을 미침을 증명
  - **다중 조건 커버리지**: 모든 조건의 가능한 조합을 테스트

## 3번 - 유닉스/리눅스 명령어

### 답: 1. pwd, 2. ls, 3. cd, 4. cp

### 풀이
- **pwd (Print Working Directory)**: 현재 작업 중인 디렉터리의 절대 경로를 화면에 출력한다.
- **ls (List)**: 현재 디렉터리 또는 지정된 디렉터리의 파일 및 하위 디렉터리 목록을 표시한다.
  - 주요 옵션: `-l`(상세 정보), `-a`(숨김 파일 포함), `-R`(재귀적 표시)
- **cd (Change Directory)**: 현재 작업 디렉터리를 다른 디렉터리로 변경한다.
  - `cd ~`(홈 디렉터리), `cd ..`(상위 디렉터리), `cd /`(루트 디렉터리)
- **cp (Copy)**: 파일이나 디렉터리를 복사한다.
  - 사용법: `cp [원본] [대상]`, 디렉터리 복사 시 `-r` 옵션 필요

## 4번 - 오류 검출 및 수정

### 답: ① Hamming ② FEC ③ BEC ④ Parity ⑤ CRC

### 풀이
- **해밍(Hamming) 코드**: 데이터 비트 사이에 검사 비트(패리티 비트)를 삽입하여 오류를 검출하고 **1비트 오류를 자동 수정**할 수 있는 코드이다. 2비트 오류까지 검출 가능하다.
- **FEC (Forward Error Correction, 전진 오류 수정)**: 수신 측이 오류를 스스로 검출하고 수정하는 방식이다. 송신 측에 재전송을 요청하지 않으므로 실시간 통신에 적합하다. 해밍 코드가 대표적이다.
- **BEC (Backward Error Correction, 후진 오류 수정)**: 수신 측이 오류를 검출하면 송신 측에 **재전송을 요청**하는 방식이다. ARQ(Automatic Repeat reQuest)가 대표적이다.
- **패리티(Parity) 검사**: 데이터 비트에 1비트의 패리티 비트를 추가하여 1의 개수가 짝수(짝수 패리티) 또는 홀수(홀수 패리티)가 되도록 하는 가장 간단한 오류 검출 방식이다. 1비트 오류만 검출 가능하며 수정은 불가능하다.
- **CRC (Cyclic Redundancy Check, 순환 중복 검사)**: 생성 다항식을 이용하여 데이터를 나눈 나머지를 검사 코드로 사용하는 방식이다. 패리티 검사보다 오류 검출 능력이 우수하며, 네트워크 통신에서 널리 사용된다.

## 7번 - C언어 연결 리스트와 비트 연산

### 답: 187

### 풀이
1. 노드 연결 구조: t3(11) → t2(7) → t1(5) → NULL
2. while 루프에서 `sum = sum * 3 + curr->x` 계산:
   - curr = t3: sum = 0 * 3 + 11 = **11**
   - curr = t2: sum = 11 * 3 + 7 = **40**
   - curr = t1: sum = 40 * 3 + 5 = **125**
   - curr = NULL → 루프 종료
3. `sum = (125 ^ 42) + 100`
   - 125 = 01111101 (2진수)
   - 42  = 00101010 (2진수)
   - XOR = 01010111 = **87**
4. sum = 87 + 100 = **187**

> 핵심 포인트: XOR(^) 연산은 같은 비트는 0, 다른 비트는 1을 반환한다.

## 8번 - Java 인터페이스

### 답: implements

### 풀이
- Java에서 클래스가 인터페이스를 구현할 때는 **implements** 키워드를 사용한다.
- `class WashingMachine implements Machine { ... }`
- Java 상속/구현 키워드 구분:
  - **extends**: 클래스 간 상속, 인터페이스 간 상속에 사용
  - **implements**: 클래스가 인터페이스를 구현할 때 사용
- 클래스는 단일 상속만 가능하지만, 인터페이스는 **다중 구현**이 가능하다.
  - 예: `class A implements B, C, D { ... }`

## 9번 - Python 리스트 처리

### 답: {0: (15, 5), 1: (10, 3), 2: (18, 5), 3: (9, 2)}

### 풀이
`enumerate(data)`로 인덱스와 리스트를 동시에 순회하며, 각 리스트의 합계와 길이를 튜플로 저장한다.

| index | lis | sum(lis) | len(lis) | result[index] |
|-------|-----|----------|----------|---------------|
| 0 | [3, 5, 2, 4, 1] | 15 | 5 | (15, 5) |
| 1 | [4, 5, 1] | 10 | 3 | (10, 3) |
| 2 | [4, 4, 1, 5, 4] | 18 | 5 | (18, 5) |
| 3 | [4, 5] | 9 | 2 | (9, 2) |

- `enumerate()` 함수는 반복 가능한 객체에 인덱스를 자동으로 부여한다.
- `sum()` 함수는 리스트 원소의 합계를, `len()` 함수는 리스트의 길이를 반환한다.
- 딕셔너리의 값으로 튜플을 저장하고 있으며, 출력 시 `{키: (값1, 값2)}` 형태로 표시된다.

## 11번 - 인증 기술

### 답: OTP (One-Time Password, 일회용 비밀번호)

### 풀이
- **OTP(One-Time Password)**는 한 번 사용하면 즉시 폐기되어 재사용이 불가능한 일회용 비밀번호이다.
- OTP 생성 방식:
  - **시간 동기화(TOTP)**: 서버와 클라이언트가 동일한 시간 기반으로 비밀번호 생성
  - **카운터 기반(HOTP)**: 이벤트 발생 횟수(카운터)를 기반으로 비밀번호 생성
  - **챌린지-응답 방식**: 서버가 난수(챌린지)를 전송하고 클라이언트가 해시 값으로 응답
- **해시 함수**를 이용하여 매번 다른 비밀번호를 생성하므로, 비밀번호 도청이나 재사용 공격에 안전하다.
- 은행 거래, 2단계 인증(2FA) 등에 널리 사용된다.

## 12번 - Java 상속과 생성자

### 답: super

### 풀이
- **super** 키워드는 자식 클래스의 생성자에서 **부모 클래스의 생성자를 호출**할 때 사용한다.
- `Square` 클래스는 `Rectangle` 클래스를 상속하며, 정사각형은 가로와 세로가 같으므로 `super(a, a)`로 호출한다.
- 실행 흐름:
  1. `new Square(5)` 호출
  2. `Square` 생성자에서 `super(5, 5)` 호출
  3. `Rectangle` 생성자에서 `this.width = 5`, `this.height = 5` 설정
- super 관련 규칙:
  - `super()`는 생성자의 **첫 번째 문장**에 위치해야 한다.
  - 명시적으로 호출하지 않으면 컴파일러가 자동으로 `super()`(기본 생성자)를 삽입한다.
  - 부모 클래스에 기본 생성자가 없으면 반드시 매개변수가 있는 `super(...)`를 명시해야 한다.

## 13번 - 인증/인가 프로토콜

### 답: OAuth

### 풀이
- **OAuth(Open Authorization)**는 사용자의 비밀번호를 제3자 서비스에 직접 제공하지 않고도 리소스 접근 권한을 위임할 수 있는 **개방형 표준 인가 프로토콜**이다.
- OAuth의 핵심 개념:
  - **Access Token**: 리소스 접근을 위한 인가 토큰 (유효기간 존재)
  - **Refresh Token**: Access Token 만료 시 새 토큰을 발급받기 위한 토큰
  - **인가 코드(Authorization Code)**: Access Token 발급을 위한 중간 코드
- 동작 흐름 (Authorization Code Grant 방식):
  1. 사용자가 클라이언트 앱에서 "구글로 로그인" 클릭
  2. 구글 인증 서버에서 사용자 인증 및 권한 동의
  3. 인가 코드 발급 → 클라이언트가 인가 코드로 Access Token 요청
  4. Access Token 발급 → 리소스 서버에 접근
- 현재 **OAuth 2.0**이 표준으로 사용되며, 소셜 로그인(구글, 카카오, 네이버 등)의 기반 기술이다.

## 14번 - 관계대수 나누기 연산

### 답:

| A |
|---|
| a1 |

### 풀이
- **나누기 연산(Division, R % S)**은 릴레이션 R에서 S의 모든 튜플과 관련된 튜플만 추출하는 연산이다.
- R % S의 결과는 R의 속성 중 S에 없는 속성에 대해, S의 **모든 값과 조합이 존재하는** 값만 반환한다.
- 즉, S의 모든 튜플을 포함하는 R의 튜플에서 S에 해당하는 속성을 제외한 나머지 속성값을 반환한다.
- 이 문제에서는 a1만이 S의 모든 튜플과 조합이 존재하므로 결과는 **a1**이다.

## 15번 - C언어 조건 연산자와 비트 연산

### 답: 0

### 풀이
1. `x=7, y=4`
2. `z = y%3<3 ? 2 : 1`
   - `y % 3` = `4 % 3` = **1**
   - `1 < 3` → **참**
   - z = **2**
3. `z = z & z >> 1`
   - `z >> 1` = `2 >> 1` = **1** (오른쪽 1비트 시프트)
   - **주의: `&` 연산자는 `>>` 보다 우선순위가 낮다** → `z & (z >> 1)`로 계산
   - `2 & 1` = `10 & 01` = **0**
   - z = **0**
4. `z = x>5 && z<=3 ? z*x : z/x`
   - `x > 5` → `7 > 5` → **참**
   - `z <= 3` → `0 <= 3` → **참**
   - 참 && 참 → **참**
   - z = `z * x` = `0 * 7` = **0**
5. 출력: **0**

> 핵심 포인트: 비트 시프트 연산자(`>>`)는 비트 AND 연산자(`&`)보다 우선순위가 높다. 또한 삼항 연산자(`?:`)는 논리 연산자(`&&`)보다 우선순위가 낮다.

## 16번 - 관계형 데이터베이스 용어

### 답: ㄱ. 튜플(Tuple), ㄴ. 인스턴스(Instance), ㄷ. 카디널리티(Cardinality)

### 풀이
- **튜플(Tuple)**: 릴레이션에서 **하나의 행(Row)**을 의미한다. 하나의 튜플은 릴레이션 스키마에 정의된 각 속성에 대한 값들의 모임이다. 레코드(Record)와 같은 개념이다.
- **인스턴스(Instance)**: 특정 시점에 릴레이션에 실제로 저장되어 있는 **데이터의 상태 전체**(모든 튜플의 집합)를 의미한다. 릴레이션의 외연(Extension)이라고도 한다.
- **카디널리티(Cardinality)**: 릴레이션에 포함된 **튜플(행)의 총 개수**를 의미한다.
- 관련 용어 정리:
  - **속성(Attribute)**: 릴레이션의 열(Column)
  - **차수(Degree)**: 속성(열)의 총 개수
  - **도메인(Domain)**: 하나의 속성이 가질 수 있는 값의 범위

## 17번 - Java enum

### 답: AB

### 풀이
1. enum `Tri` 정의:
   - `A("A")` → code = "A", ordinal = 0
   - `B("AB")` → code = "AB", ordinal = 1
   - `C("ABC")` → code = "ABC", ordinal = 2
2. `Tri.A.name()` → **"A"** (enum 상수의 이름을 문자열로 반환)
3. `Tri.A.name().length()` → `"A".length()` → **1**
4. `Tri.values()` → `{A, B, C}` (모든 enum 상수의 배열)
5. `Tri.values()[1]` → **Tri.B**
6. `t = Tri.B`
7. `t.code()` → Tri.B의 code 필드 → **"AB"**
8. 출력: **AB**

> 핵심 포인트:
> - `name()`: enum 상수의 선언된 이름을 문자열로 반환 (Tri.A.name() = "A")
> - `values()`: 모든 enum 상수를 선언 순서대로 배열로 반환
> - `ordinal()`: enum 상수의 순서(0부터 시작)를 반환
> - enum은 사용자 정의 필드와 메서드를 가질 수 있다.

## 18번 - 접근통제 모델

### 답: ㄱ. MAC, ㄴ. RBAC, ㄷ. DAC

### 풀이
- **MAC (Mandatory Access Control, 강제적 접근통제)**
  - **중앙 관리자**가 보안 정책을 일괄적으로 설정하고 관리한다.
  - 주체와 객체에 보안 등급(Security Level)을 부여하여 접근을 통제한다.
  - 군사/정부 기관 등 높은 보안이 요구되는 환경에서 사용한다.
  - 대표 모델: Bell-LaPadula(기밀성), Biba(무결성)

- **RBAC (Role-Based Access Control, 역할 기반 접근통제)**
  - 사용자에게 직접 권한을 부여하지 않고, **역할(Role)**을 부여한 뒤 역할에 권한을 매핑한다.
  - 조직의 직무 분리(Separation of Duty)를 반영할 수 있다.
  - 사용자 수가 많은 기업 환경에서 관리가 효율적이다.
  - MAC과 DAC의 단점을 보완한 방식이다.

- **DAC (Discretionary Access Control, 임의적 접근통제)**
  - 자원의 **소유자**가 자신의 판단에 따라 다른 사용자에게 권한을 자유롭게 부여/회수한다.
  - ACL(Access Control List)을 기반으로 구현된다.
  - 유연하지만, 소유자의 판단에 의존하므로 보안이 상대적으로 취약하다.
  - 대표 시스템: UNIX/Linux 파일 시스템의 chmod 명령어

## 19번 - 테스트케이스 구성요소

### 답: ㄱ. 테스트 조건 / ㄹ. 테스트 데이터 / ㅁ. 예상 결과

### 풀이
- **테스트 조건(Test Condition)**: 테스트를 수행하기 위해 사전에 충족되어야 하는 **전제 조건**이다. 시스템 환경 설정, 초기 데이터 상태 등이 포함된다.
- **테스트 데이터(Test Data)**: 테스트를 실행하기 위해 시스템에 입력하는 **입력값**이다. 정상 데이터, 경계값, 비정상 데이터 등을 포함한다.
- **예상 결과(Expected Result)**: 테스트 수행 후 **기대되는 출력값이나 시스템의 상태**이다. 실제 결과와 비교하여 테스트 통과/실패를 판단하는 기준이 된다.
- 테스트케이스의 주요 구성요소: 테스트 ID, 테스트 항목, 테스트 조건, 테스트 절차, 테스트 데이터, 예상 결과, 실제 결과, 판정 등

## 20번 - SQL 쿼리

### 답: 4

### 풀이
- `SELECT count(col2) FROM A WHERE col1 IN (2, 3) OR col2 IN (3, 5)`
- **WHERE 조건 분석**:
  - `col1 IN (2, 3)`: col1 값이 2 또는 3인 행을 선택
  - `col2 IN (3, 5)`: col2 값이 3 또는 5인 행을 선택
  - **OR** 연산이므로 두 조건 중 하나라도 만족하면 선택
- **count(col2)**는 선택된 행 중 col2 값이 **NULL이 아닌 행의 수**를 반환한다.
- 조건을 만족하는 행 중 col2가 NULL이 아닌 행이 **4개**이므로 결과는 **4**이다.

> 핵심 포인트:
> - `count(컬럼명)`: 해당 컬럼이 NULL이 아닌 행의 수를 반환
> - `count(*)`: NULL 포함 모든 행의 수를 반환
> - `IN` 연산자: 지정된 값 목록 중 하나와 일치하는지 확인
