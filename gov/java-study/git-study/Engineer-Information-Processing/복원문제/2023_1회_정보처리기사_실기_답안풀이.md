## 1번 - Java static 변수

### 코드

```java
class Static{
    public int a = 20;
    static int b = 0;
}
public class Main{
    public static void main(String[] args) {
        int a;
        a = 10;
        Static.b = a;
        Static st = new Static();
        System.out.println(Static.b++);
        System.out.println(st.b);
        System.out.println(a);
        System.out.print(st.a);
    }
}
```

### 답

```
10
11
10
20
```

### 풀이

1. 지역변수 `a = 10`으로 설정한다.
2. `Static.b = a` → static 변수 `b`에 10을 대입한다.
3. `Static.b++` → **후위 증가 연산자**이므로 현재 값 **10을 먼저 출력**한 뒤, `b`가 11로 증가한다.
4. `st.b` → static 변수는 클래스 전체에서 공유되므로 증가된 값 **11**이 출력된다.
5. `a` → main 메서드의 지역변수이므로 그대로 **10**이 출력된다.
6. `st.a` → 인스턴스 변수로 초기값 **20**이 출력된다.

> **핵심 포인트**: static 변수는 모든 인스턴스가 공유하며, 후위 증가 연산자(`b++`)는 현재 값을 사용한 후 증가시킨다.

## 2번 - C언어 문자열 포인터

### 코드

```c
#include <stdio.h>
int main(){
    char a[] = "Art";
    char* p = NULL;
    p = a;
    printf("%s\n", a);
    printf("%c\n", *p);
    printf("%c\n", *a);
    printf("%s\n", p);
    for(int i = 0; a[i] != '\0'; i++)
        printf("%c", a[i]);
}
```

### 답

```
Art
A
A
Art
Art
```

### 풀이

1. `char a[] = "Art"` → 배열 a에 문자열 "Art"가 저장된다.
2. `p = a` → 포인터 p가 배열 a의 시작 주소를 가리킨다.
3. `printf("%s\n", a)` → 배열 이름은 시작 주소이므로 문자열 전체 **"Art"** 출력.
4. `printf("%c\n", *p)` → p가 가리키는 첫 번째 문자 **'A'** 출력.
5. `printf("%c\n", *a)` → 배열의 첫 번째 문자 **'A'** 출력.
6. `printf("%s\n", p)` → p부터 문자열 전체 **"Art"** 출력.
7. for문으로 한 글자씩 출력하면 **"Art"** 출력.

> **핵심 포인트**: 배열 이름과 포인터는 모두 시작 주소를 나타내며, `%s`는 문자열 전체를, `%c`는 단일 문자를 출력한다.

## 3번 - C언어 문자열 비교

### 코드

```c
#include <stdio.h>
int main(){
    char* a = "qwer";
    char* b = "qwtety";
    for(int i = 0; a[i] != '\0'; i++){
        for(int j = 0; b[j] != '\0'; j++){
            if(a[i] == b[j]) printf("%c", a[i]);
        }
    }
}
```

### 답

```
qwe
```

### 풀이

a의 각 문자를 b의 모든 문자와 비교하여 일치하면 출력한다.

| a[i] | b에서 탐색 | 일치 여부 | 출력 |
|------|-----------|----------|------|
| q | q**w**tety → q에서 일치 | 1회 일치 | q |
| w | q**w**tety → w에서 일치 | 1회 일치 | w |
| e | qwt**e**ty → e에서 일치 | 1회 일치 | e |
| r | qwtety → 일치 없음 | 0회 | (없음) |

결과: **qwe**

> **핵심 포인트**: 이중 for문으로 a의 각 문자가 b에 존재하는지 확인하는 구조이다. 일치할 때마다 출력하므로 b에 같은 문자가 여러 개 있으면 중복 출력될 수 있다.

## 9번 - C언어 이진수에서 십진수 변환

### 코드

```c
#include <stdio.h>
int main() {
    int input = 101110;
    int di = 1;
    int sum = 0;
    while (1) {
        if (input == 0) break;
        else {
            sum = sum + (input % 10) * di;
            di = di * 2;
            input = input / 10;
        }
    }
    printf("%d", sum);
    return 0;
}
```

### 답

**(a) %**
**(b) 10**

출력 결과: **46**

### 풀이

`input % 10`은 input의 마지막 자릿수를 추출하는 연산이다. 이를 통해 이진수의 각 자릿수를 하나씩 꺼내어 10진수로 변환한다.

| 반복 | input | input % 10 | di | sum |
|------|-------|-----------|-----|-----|
| 1 | 101110 | 0 | 1 | 0 |
| 2 | 10111 | 1 | 2 | 2 |
| 3 | 1011 | 1 | 4 | 6 |
| 4 | 101 | 1 | 8 | 14 |
| 5 | 10 | 0 | 16 | 14 |
| 6 | 1 | 1 | 32 | 46 |

101110(2) = 0×1 + 1×2 + 1×4 + 1×8 + 0×16 + 1×32 = **46**

## 10번 - ICMP 공격

### 문제

> TCP/IP의 IP 패킷 처리 알림 프로토콜을 이용하며, ping 명령어를 연속적으로 전송하여 공격하는 방식이다.

### 답

**ICMP Flooding** (Ping Flooding / Ping of Death)

### 풀이

ICMP(Internet Control Message Protocol)는 IP 패킷 처리 과정에서 발생하는 오류를 보고하거나 네트워크 상태를 진단하는 프로토콜이다. **ICMP Flooding** 공격은 대량의 ICMP Echo Request(ping) 패킷을 목표 시스템에 지속적으로 전송하여 시스템 자원을 고갈시키는 **DoS(서비스 거부) 공격**의 일종이다.

- **Ping Flooding**: 대량의 ping 요청을 지속적으로 전송
- **Ping of Death**: 규격 이상의 큰 ICMP 패킷을 전송하여 시스템 오류 유발

## 11번 - 디자인 패턴

### 문제

> 다른 객체에 대한 접근을 제어하기 위해 대리 역할을 하며, 실제 객체 호출 시 행위를 가로채는 패턴이다.

### 답

**Proxy 패턴**

### 풀이

Proxy 패턴은 **구조 패턴(Structural Pattern)** 중 하나로, 실제 객체에 대한 대리자(대변인) 역할을 하는 객체를 두어 접근을 제어하는 패턴이다.

- **원격 프록시(Remote Proxy)**: 원격 객체에 대한 로컬 대리자 역할
- **가상 프록시(Virtual Proxy)**: 무거운 객체의 생성을 지연시킴
- **보호 프록시(Protection Proxy)**: 접근 권한을 제어

클라이언트는 Proxy 객체를 통해 실제 객체에 접근하며, Proxy가 요청을 가로채어 전처리/후처리를 수행할 수 있다.

## 12번 - 데이터베이스 용어

### 답

**(1) 튜플(Tuple)**
> 순서가 있는 열거이며, 중복을 허용하고, 여러 객체를 포함한다.

**(2) 릴레이션 인스턴스(Instance)**
> 특정 시점의 릴레이션 내용이며, 저장된 데이터 전체를 의미한다.

**(3) 카디널리티(Cardinality)**
> 유니크한 값의 개수를 나타내며, 중복 수치의 지표이다.

### 풀이

- **튜플(Tuple)**: 릴레이션에서 하나의 행(Row)을 의미한다. 속성(Attribute)들의 값으로 구성되며, 릴레이션 내에서 각 튜플은 유일해야 한다.
- **릴레이션 인스턴스(Instance)**: 어느 한 시점에 릴레이션에 실제로 저장되어 있는 튜플들의 집합이다. 데이터의 삽입, 삭제, 갱신에 따라 시간에 따라 변할 수 있다.
- **카디널리티(Cardinality)**: 특정 컬럼에서 유니크한(고유한) 값의 수를 의미한다. 카디널리티가 높을수록 중복도가 낮고, 인덱스 효율이 좋다.

## 14번 - Java 정렬 코드 빈칸

### 코드

```java
public class Sort {
    public static void swap(int[] arr, int idx1, int idx2){
        int temp = arr[idx1];
        arr[idx1] = arr[idx2];
        arr[(1)] = temp;
    }
    // Usort 메서드에서 재귀호출 Usort(item, (2))
}
```

### 답

**(1) idx2**

**(2) nx**

### 풀이

- **(1) idx2**: swap 메서드는 배열의 두 원소를 교환하는 함수이다. `temp`에 `arr[idx1]`을 저장하고, `arr[idx1]`에 `arr[idx2]`를 대입한 후, `arr[idx2]`에 `temp`를 넣어야 교환이 완성된다.
- **(2) nx**: 정렬 알고리즘에서 재귀 호출 시 다음(next) 인덱스인 `nx`를 매개변수로 전달하여 정렬 범위를 조정한다.

## 15번 - Python Set 연산

### 코드

```python
a = {'한국', '중국', '일본'}
a.add('베트남')
a.add('중국')
a.remove('일본')
a.update({'홍콩', '한국', '태국'})
print(a)
```

### 답

```
{'한국', '중국', '베트남', '홍콩', '태국'}
```

(Set은 순서가 없으므로 출력 순서는 다를 수 있다)

### 풀이

Python의 Set(집합)은 **중복을 허용하지 않고**, **순서가 없는** 자료구조이다.

| 연산 | 실행 후 a의 상태 |
|------|----------------|
| 초기값 | {'한국', '중국', '일본'} |
| `a.add('베트남')` | {'한국', '중국', '일본', '베트남'} |
| `a.add('중국')` | {'한국', '중국', '일본', '베트남'} ← 이미 존재하므로 변화 없음 |
| `a.remove('일본')` | {'한국', '중국', '베트남'} |
| `a.update({'홍콩', '한국', '태국'})` | {'한국', '중국', '베트남', '홍콩', '태국'} ← '한국'은 이미 존재 |

> **핵심 포인트**: `add()`는 이미 존재하는 원소를 추가해도 무시되고, `update()`는 합집합 연산과 동일하다.

## 16번 - SQL GROUP BY HAVING

### 문제

> 성적 테이블에서 과목별 평균 점수가 90점 이상인 과목의 과목이름, 최소점수, 최대점수를 검색하시오.

### 답

```sql
SELECT 과목이름, MIN(점수) AS 최소점수, MAX(점수) AS 최대점수
FROM 성적
GROUP BY 과목이름
HAVING AVG(점수) >= 90;
```

### 풀이

- `GROUP BY 과목이름`: 과목이름별로 그룹을 만든다.
- `HAVING AVG(점수) >= 90`: 그룹화된 결과에서 평균 점수가 90 이상인 그룹만 필터링한다.
- `MIN(점수)`, `MAX(점수)`: 각 그룹의 최소, 최대 점수를 구한다.

> **WHERE vs HAVING**: `WHERE`는 그룹화 이전에 개별 행을 필터링하고, `HAVING`은 그룹화 이후에 그룹을 필터링한다. 집계 함수(AVG, SUM 등)를 조건으로 사용할 때는 반드시 `HAVING`을 사용해야 한다.

```java
        Vehicle obj = new Car("Spark");
        System.out.println(obj.getName());
    }
}
```

### 답

```
Vehicle name: Spark
```

### 풀이

1. `new Car("Spark")` → Car 생성자 호출 → `name = super.name = "Spark"` → Vehicle의 name 필드에 "Spark" 저장.
2. `obj.getName()` → **매개변수가 없는** `getName()` 메서드 호출.
3. Car 클래스에는 `getName(String val)`과 `getName(byte val[])`은 있지만, **매개변수 없는 `getName()`은 오버라이딩하지 않았다**.
4. 따라서 Vehicle의 `getName()` 메서드가 실행된다.
5. `"Vehicle name: " + name` → name은 "Spark"이므로 → **"Vehicle name: Spark"** 출력.

> **핵심 포인트**: 메서드 오버라이딩은 매개변수 시그니처가 완전히 동일해야 한다. `getName()`과 `getName(String val)`은 서로 다른 메서드이다.

## 18번 - 데이터베이스 스키마

### 답

**(1) 외부 스키마 (External Schema)**
> 사용자 입장에서의 논리적 구조이며, 서브스키마라고도 한다.

**(2) 개념 스키마 (Conceptual Schema)**
> 전체 데이터베이스의 논리적 구조이며, 하나만 존재한다.

**(3) 내부 스키마 (Internal Schema)**
> 물리적 저장 방법을 정의하며, 시스템 프로그래머가 관리한다.

### 풀이

데이터베이스의 **3단계 스키마 구조**(ANSI/SPARC 아키텍처):

| 스키마 | 관점 | 설명 |
|--------|------|------|
| **외부 스키마** | 사용자 | 개별 사용자가 보는 데이터의 논리적 구조. 서브스키마라고도 하며, 여러 개 존재 가능 |
| **개념 스키마** | 조직 전체 | 전체 DB의 논리적 구조를 정의. 단 하나만 존재하며, DBA가 관리 |
| **내부 스키마** | 저장 장치 | 데이터의 물리적 저장 구조를 정의. 인덱스, 레코드 배치 등을 기술 |

- **외부/개념 매핑**: 논리적 독립성
- **개념/내부 매핑**: 물리적 독립성

## 19번 - 분기 커버리지

### 문제

> 제어 흐름 그래프에서 분기 커버리지(Branch Coverage)를 만족하는 경로를 쓰시오.

### 답

**1234561, 124567**

### 풀이

분기 커버리지(Branch Coverage)는 프로그램의 모든 분기(조건문의 참/거짓)를 최소 한 번씩 실행하는 테스트 커버리지 기준이다.

- **경로 1234561**: 조건을 만족하여 분기의 **참(True)** 경로를 실행하는 경우.
- **경로 124567**: 조건을 만족하지 않아 분기의 **거짓(False)** 경로를 실행하는 경우.

이 두 경로를 실행하면 모든 분기의 참/거짓을 커버할 수 있으므로 **분기 커버리지 100%**를 달성한다.

> **참고**: 분기 커버리지는 구문 커버리지(Statement Coverage)보다 강하고, 조건 커버리지(Condition Coverage)와는 독립적인 관계이다.

## 20번 - Java 생성자 체이닝

### 코드

```java
class Parent {
    int x = 100;
    Parent() { this(500); }
    Parent(int x) { this.x = x; }
    int getX() { return x; }
}
class Child extends Parent {
    int x = 4000;
    Child() { this(5000); }
    Child(int x) { this.x = x; }
}
public class Main {
    public static void main(String[] args) {
        Child obj = new Child();
        System.out.println(obj.getX());
    }
}
```

### 답

```
500
```

### 풀이

생성자 호출 순서를 단계별로 추적하면 다음과 같다:

```
1. new Child()
2. → Child() 생성자 호출
3. → this(5000) → Child(int x) 생성자 호출
4. → 암묵적 super() → Parent() 생성자 호출
5. → this(500) → Parent(int x) 생성자 호출
6. → this.x = 500 → Parent의 x = 500
7. → Child(int x)로 돌아옴 → this.x = 5000 → Child의 x = 5000
```

최종 상태:
- **Parent.x = 500**
- **Child.x = 5000**

`obj.getX()`는 **Parent 클래스에 정의된 메서드**이며, Child에서 오버라이딩하지 않았다. Parent의 `getX()`는 `return x`에서 **Parent의 x 필드**를 참조하므로 **500**을 반환한다.

> **핵심 포인트**: Parent와 Child에 동일한 이름의 인스턴스 변수 `x`가 각각 존재한다(필드 은닉, Field Hiding). 메서드는 오버라이딩되지만 필드는 오버라이딩되지 않으므로, `getX()`가 참조하는 `x`는 메서드가 정의된 클래스(Parent)의 필드이다.
