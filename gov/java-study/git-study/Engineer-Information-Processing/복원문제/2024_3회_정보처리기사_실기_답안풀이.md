## 1번 - Java 코드 출력값

### 답: `OOAAA`

### 풀이

```java
public class Main{
  static String[] s = new String[3];
  static void func(String[]s, int size){
    for(int i=1; i<size; i++){
      if(s[i-1].equals(s[i])){
        System.out.print("O");
      }else{
        System.out.print("N");
      }
    }
    for (String m : s){
      System.out.print(m);
    }
  }
  public static void main(String[] args){
    s[0] = "A";
    s[1] = "A";
    s[2] = new String("A");
    func(s, 3);
  }
}
```

- `equals()` 메서드는 문자열의 **내용(값)**을 비교한다. (`==`는 참조 주소를 비교)
- `s[0]`과 `s[1]`은 모두 리터럴 `"A"`이므로 `s[0].equals(s[1])` = **true** → `"O"` 출력
- `s[1]`과 `s[2]`는 `new String("A")`로 생성했지만, `equals()`는 내용을 비교하므로 `s[1].equals(s[2])` = **true** → `"O"` 출력
- 첫 번째 for문 결과: `"OO"`
- 두 번째 for-each문에서 `s[0]`, `s[1]`, `s[2]` 모두 `"A"` → `"AAA"` 출력
- **최종 출력: `OOAAA`**

## 2번 - Python 코드 출력값

### 답: `3`

### 풀이

```python
def func(lst):
  for i in range(len(lst) //2):
    lst[i], lst[-i-1] = lst[-i-1], lst[i]
lst = [1,2,3,4,5,6]
func(lst)
print(sum(lst[::2]) - sum(lst[1::2]))
```

1. `func(lst)` 실행: `len(lst)//2 = 3`이므로 `i = 0, 1, 2`
   - `i=0`: `lst[0]`과 `lst[-1]` 교환 → `[6,2,3,4,5,1]`
   - `i=1`: `lst[1]`과 `lst[-2]` 교환 → `[6,5,3,4,2,1]`
   - `i=2`: `lst[2]`과 `lst[-3]` 교환 → `[6,5,4,3,2,1]`
2. 리스트가 **뒤집혀서** `[6,5,4,3,2,1]`이 됨
3. `lst[::2]` = 짝수 인덱스 요소 = `[6, 4, 2]` → `sum = 12`
4. `lst[1::2]` = 홀수 인덱스 요소 = `[5, 3, 1]` → `sum = 9`
5. **최종 출력: `12 - 9 = 3`**

## 5번 - 네트워크 공격

### 답: 스머프(Smurf) 또는 스머핑(Smurfing)

### 풀이

- **스머프 공격(Smurf Attack)**은 ICMP 프로토콜을 악용한 DDoS 공격의 일종이다.
- 공격자가 출발지 IP를 **피해자의 IP로 위조**하여 브로드캐스트 주소로 ICMP Echo Request를 전송한다.
- 네트워크 내 모든 호스트가 피해자에게 ICMP Echo Reply를 보내게 되어 **대량의 트래픽이 피해자에게 집중**된다.
- IP/ICMP를 악용하며, 다량의 ICMP Echo Reply를 이용하는 것이 핵심 특징이다.

## 6번 - GoF 디자인 패턴

### 답: 행위(Behavioral) 패턴

### 풀이

GoF(Gang of Four) 디자인 패턴은 3가지로 분류된다:

| 분류 | 설명 | 대표 패턴 |
|------|------|----------|
| **생성(Creational)** | 객체 생성 과정을 추상화 | Factory Method, Singleton, Builder, Prototype, Abstract Factory |
| **구조(Structural)** | 클래스/객체 조합으로 더 큰 구조 형성 | Adapter, Bridge, Composite, Decorator, Facade, Proxy |

- **행위 패턴**은 클래스나 객체들이 서로 **상호작용하는 방법**을 정의하고, **알고리즘을 캡슐화**한다.
- Chain of Responsibility, Command, Observer 등이 해당된다.

## 7번 - C언어 코드 출력값

### 답: `20`

### 풀이

```c
#include <stdio.h>
int func(){
  static int x = 0;
  x += 2;
  return x;
}
int main(){
  int x = 1;
  int sum = 0;
  for(int i=0; i<4; i++) {
    x++;
    sum += func();
  }
  printf("%d", sum);
  return 0;
}
```

- `func()` 내부의 `static int x = 0`은 **정적 변수**로, 함수 호출이 끝나도 값이 유지된다.
- `main()`의 `x`와 `func()`의 `x`는 **별개의 변수**이다.

| 반복 | func()의 static x | func() 반환값 | sum |
|------|-------------------|--------------|-----|
| i=0 | 0+2=2 | 2 | 2 |
| i=1 | 2+2=4 | 4 | 6 |
| i=2 | 4+2=6 | 6 | 12 |
| i=3 | 6+2=8 | 8 | 20 |

## 9번 - URL 구조

### 답: `43125`

### 풀이

URL의 구조는 다음과 같다:

```
scheme://authority/path?query#fragment
```

- **scheme**: 리소스에 접근하기 위한 프로토콜 (예: http, https, ftp)
- **authority**: 호스트 정보 (예: www.example.com:8080)
- **path**: 서버 내 리소스 경로 (예: /api/users)
- **query**: 추가 데이터 전달 (예: ?id=1&name=test)
- **fragment**: 문서 내 특정 위치 (예: #section1)

순서 매칭: scheme=1, authority=2, path=3, query=4, fragment=5 → **43125**

```python
def func(value):
    if type(value) == type(100):        # int 타입 체크
        return 100
    elif type(value) == type(""):       # str 타입 체크
        return len(value)
    else:                               # 그 외 타입
```

## 11번 - Java 코드 출력값

### 답: `52`

### 풀이

```java
public class Main{
  public static void main(String[] args){
    Base a = new Derivate();
    Derivate b = new Derivate();
    System.out.print(a.getX() + a.x + b.getX() + b.x);
  }
}
class Base{
  int x = 3;
  int getX(){ return x * 2; }
}
class Derivate extends Base{
  int x = 7;
  int getX(){ return x * 3; }
}
```

핵심 개념: **메서드는 동적 바인딩(오버라이딩)**, **필드는 정적 바인딩**

- `a.getX()`: `a`의 실제 객체는 `Derivate`이므로 `Derivate.getX()` 호출 → `7 * 3 = 21`
- `a.x`: `a`의 선언 타입이 `Base`이므로 `Base.x` 참조 → `3` (필드는 정적 바인딩)
- `b.getX()`: `b`는 `Derivate` 타입이므로 `Derivate.getX()` 호출 → `7 * 3 = 21`
- `b.x`: `b`의 선언 타입이 `Derivate`이므로 `Derivate.x` 참조 → `7`
- **최종 출력: `21 + 3 + 21 + 7 = 52`**

## 12번 - C언어 연결리스트 출력값

### 답: `312`

### 풀이

```c
struct Node { int value; struct Node* next; };
void func(struct Node* node){
  while(node != NULL && node->next != NULL){
    int t = node->value;
    node->value = node->next->value;
    node->next->value = t;
    node = node->next->next;
  }
}
```

1. **초기 연결 상태**: `n1(1) → n3(3) → n2(2) → NULL`
2. `func(&n1)` 실행:
   - `node = &n1`, `node->next = &n3` (둘 다 NULL이 아님)
   - `n1`과 `n3`의 value 교환: `n1(3) → n3(1) → n2(2)`
   - `node = node->next->next = &n2`
   - `node = &n2`, `node->next = NULL` → 반복 종료
3. **순회 출력: `3 → 1 → 2` = `312`**

## 13번 - 테스트 커버리지

### 답:
1. **문장 커버리지 (Statement Coverage)**
2. **분기 커버리지 (Branch Coverage)** 또는 결정 커버리지(Decision Coverage)
3. **조건 커버리지 (Condition Coverage)**

### 풀이

| 커버리지 | 설명 |
|---------|------|
| **문장 커버리지** | 프로그램의 모든 문장(statement)을 최소 1번 실행 |
| **분기 커버리지** | 모든 분기(branch)의 참/거짓 경로를 최소 1번 실행 |
| **조건 커버리지** | 각 개별 조건식이 참과 거짓을 모두 한 번씩 평가 |

- 문장 커버리지는 가장 약한 커버리지 기준이다.
- 분기 커버리지는 문장 커버리지를 포함한다.
- 조건 커버리지는 각 개별 조건을 독립적으로 평가하므로, 분기 커버리지와는 독립적인 관계이다.

## 16번 - C언어 포인터 코드 출력값

### 답: `1`

### 풀이

```c
#include <stdio.h>
void func(int** arr, int size){
  for(int i=0; i<size; i++){
    *(*arr + i) = (*(*arr + i) + i) % size;
  }
}
int main(){
  int arr[] = {3, 1, 4, 1, 5};
  int* p = arr;
  int** pp = &p;
  int num = 6;
  func(pp, 5);
  num = arr[2];
  printf("%d", num);
  return 0;
}
```

- `*arr`은 포인터 `p`의 값, 즉 배열 `arr`의 시작 주소이다.
- `*(*arr + i)`는 `arr[i]`와 동일하다.
- `func` 실행: 각 `arr[i] = (arr[i] + i) % 5`

| i | 기존 arr[i] | (arr[i] + i) % 5 | 결과 |
|---|------------|-------------------|------|
| 0 | 3 | (3 + 0) % 5 = 3 | 3 |
| 1 | 1 | (1 + 1) % 5 = 2 | 2 |
| 2 | 4 | (4 + 2) % 5 = 1 | 1 |
| 3 | 1 | (1 + 3) % 5 = 4 | 4 |
| 4 | 5 | (5 + 4) % 5 = 4 | 4 |

- `num = arr[2] = 1`
- **최종 출력: `1`**

## 18번 - Java 예외 처리 출력값

### 답: `101`

### 풀이

```java
public class ExceptionHandling {
  public static void main(String[] args) {
    int sum = 0;
    try {
      func();
    } catch (NullPointerException e) {
      sum = sum + 1;
    } catch (Exception e) {
      sum = sum + 10;
    } finally {
      sum = sum + 100;
    }
    System.out.print(sum);
  }
  static void func() throws Exception {
    throw new NullPointerException();
  }
}
```

1. `func()` 호출 → `NullPointerException` 발생
2. `catch (NullPointerException e)` 블록에서 처리 → `sum = 0 + 1 = 1`
3. `catch (Exception e)` 블록은 **이미 처리되었으므로 건너뜀** (상위 예외 클래스)
4. `finally` 블록은 **항상 실행** → `sum = 1 + 100 = 101`
5. **최종 출력: `101`**

> **참고**: `NullPointerException`은 `RuntimeException`의 하위 클래스이고, `RuntimeException`은 `Exception`의 하위 클래스이다. catch 블록은 위에서부터 순서대로 매칭되므로, 더 구체적인 예외를 먼저 작성해야 한다.


핵심 개념: **제네릭 타입 소거(Type Erasure)**

- Java의 제네릭은 **컴파일 시점에 타입이 소거**된다.
- `Collection<T>`의 `T`는 컴파일 후 **`Object`로 변환**된다.
- `value`의 컴파일 타입은 `Object`이다.
- **메서드 오버로딩은 컴파일 시점에 결정**(정적 바인딩)되므로, `print(value)`는 `print(Object a)`를 호출한다.
- 실행 시점에 `value`가 실제로 `Integer`이더라도, 이미 컴파일 시점에 `print(Object)`로 결정되었다.
- **최종 출력: `B0`**

## 20번 - 네트워크 용어

### 답: 애드혹 네트워크 (Ad-hoc Network)

### 풀이

- **애드혹 네트워크(Ad-hoc Network)**는 고정된 인프라(액세스 포인트, 라우터 등) 없이 **임시로 구성**되는 네트워크이다.
- 노드들이 **무선 통신으로 직접(peer-to-peer)** 연결된다.
- **중앙 관리 장치가 없으며**, 각 노드가 라우터 역할을 수행할 수 있다.
- 주요 활용 분야:
  - **긴급 상황** (재난, 군사 작전)
  - 임시 회의나 이벤트
  - 센서 네트워크
- MANET(Mobile Ad-hoc Network)은 이동 노드로 구성된 애드혹 네트워크이다.
