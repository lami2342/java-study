## 1번 - Java 프로그래밍 (싱글톤 패턴)

```java
class Connection {
    private static Connection _inst = null;
    private int count = 0;
    static public Connection get() {
        if(_inst == null) {
            _inst = new Connection();
            return _inst;
        }
        return _inst;
    }
    public void count() { count++; }
    public int getCount() { return count; }
}
public class main {
    public static void main(String[] args) {
        Connection conn1 = Connection.get();
        conn1.count();
        Connection conn2 = Connection.get();
        conn2.count();
        Connection conn3 = Connection.get();
        conn3.count();
        conn1.count();
        System.out.print(conn1.getCount());
    }
}
```

### 답: `4`

### 풀이:
이 코드는 **싱글톤(Singleton) 패턴**을 구현하고 있다.

1. `Connection.get()`은 `_inst`가 `null`일 때만 새 인스턴스를 생성하고, 이후에는 동일한 인스턴스를 반환한다.
2. 따라서 `conn1`, `conn2`, `conn3`은 **모두 같은 객체**를 참조한다.
3. `count()` 메서드가 총 **4번** 호출된다:
   - `conn1.count()` → count = 1
   - `conn2.count()` → count = 2
   - `conn3.count()` → count = 3
   - `conn1.count()` → count = 4
4. `conn1.getCount()` → **4** 출력

## 2번 - C언어 프로그래밍 (조건 연산자)

```c
#include <stdio.h>
int main() {
    int v1 = 0, v2 = 35, v3 = 29;
    if(v1 > v2 ? v2 : v1) {
        v2 = v2 << 2;
    }else{
        v3 = v3 << 2;
    }
    printf("%d", v2+v3);
}
```

### 답: `151`

### 풀이:
1. 삼항 연산자 `v1 > v2 ? v2 : v1` 분석:
   - `v1 > v2` → `0 > 35` → **거짓(false)**
   - 거짓이므로 `v1` 값인 **0**이 선택됨
2. `if(0)` → **거짓**이므로 `else` 블록 실행
3. `v3 = v3 << 2` → `29 << 2` → 29 x 4 = **116**
4. `v2 + v3` = 35 + 116 = **151**

> **참고**: `<<` 는 왼쪽 시프트 연산자로, `<< 2`는 2^2 = 4를 곱하는 것과 같다.

## 3번 - 소프트웨어 공학 (응집도 순서)

### 답: `ㄱ, ㄴ, ㄹ, ㄷ` (기능 > 교환 > 시간 > 우연)

### 풀이:
응집도(Cohesion)는 모듈 내부 요소들이 서로 관련되어 있는 정도를 나타낸다. 높은 순서대로:

| 순위 | 응집도 | 설명 |
|------|--------|------|
| 1 (최고) | **기능적 응집도** | 모듈 내 모든 요소가 단일 기능 수행 |
| 2 | 순차적 응집도 | 한 요소의 출력이 다음 요소의 입력 |
| 3 | **교환(통신적) 응집도** | 동일 입출력 데이터를 사용하는 요소들 |
| 4 | 절차적 응집도 | 특정 순서로 실행되는 요소들 |
| 5 | **시간적 응집도** | 같은 시간에 실행되는 요소들 |
| 6 | 논리적 응집도 | 유사한 성격의 요소들 |
| 7 (최저) | **우연적 응집도** | 관련 없는 요소들이 우연히 모임 |

따라서 보기 중 높은 순서: **기능(ㄱ) > 교환(ㄴ) > 시간(ㄹ) > 우연(ㄷ)**

## 4번 - C언어 프로그래밍 (문자열 역순)

```c
#include <stdio.h>
#include <string.h>
void reverse(char* str){
    int len = strlen(str);
    char temp;
    char*p1 = str;
    char*p2 = str + len - 1;
    while(p1<p2){
        temp = *p1;
        *p1 = *p2;
        *p2 = temp;
        p1++;
        p2--;
    }
}
int main(int argc, char* argv[]){
    char str[100] = "ABCDEFGH";
    reverse(str);
    int len = strlen(str);
    for(int i=1; i<len; i+=2){
        printf("%c",str[i]);
    }
    printf("\n");
    return 0;
}
```

### 답: `GECA`

### 풀이:
1. `reverse("ABCDEFGH")` 실행 → 문자열이 **"HGFEDCBA"** 로 뒤집힘
2. `for(int i=1; i<len; i+=2)` → 인덱스 **1, 3, 5, 7** 의 문자를 출력
   - 인덱스 0: H
   - 인덱스 1: **G**
   - 인덱스 2: F
   - 인덱스 3: **E**
   - 인덱스 4: D
   - 인덱스 5: **C**
   - 인덱스 6: B
   - 인덱스 7: **A**
3. 출력 결과: **GECA**

## 7번 - 네트워크 (라우팅 프로토콜)

### 답: `OSPF (Open Shortest Path First)`

### 풀이:
- **링크 상태(Link State) 프로토콜**: 네트워크 전체 토폴로지 정보를 유지하여 최적 경로를 계산
- **내부 게이트웨이 프로토콜(IGP)**: 하나의 AS(자율 시스템) 내부에서 사용
- **Dijkstra 알고리즘**: 최단 경로를 구하는 알고리즘

이 세 가지 특성을 모두 만족하는 프로토콜은 **OSPF**이다.

> **참고**: RIP은 거리 벡터 방식이고, BGP는 외부 게이트웨이 프로토콜(EGP)이다.

## 11번 - C언어 프로그래밍 (구조체 및 함수)

```c
#include <stdio.h>
typedef struct{
    int accNum;
    double bal;
}BankAcc;

double sim_pow(double base, int year){
    int i;
    double r = 1.0;
    for(i=0; i<year; i++){
        r = r*base;
    }
    return r;
}

void initAcc(BankAcc *acc, int x, double y){
    acc->accNum = x;
    acc->bal = y;
}

void xxx(BankAcc *acc, double *en){
    if (*en > 0 && *en < acc->bal) {
        acc->bal = acc->bal - *en;
    }else{
        acc->bal = acc->bal + *en;
    }
}

void yyy(BankAcc *acc){
    acc->bal = acc->bal * sim_pow((1+0.1), 3);
}

int main(){
    BankAcc myAcc;
    initAcc(&myAcc, 9981, 2200.0);
    double amount = 100.0;
    xxx(&myAcc, &amount);
    yyy(&myAcc);
    printf("%d and %.2f", myAcc.accNum, myAcc.bal);
    return 0;
}
```

### 답: `9981 and 2795.10`

### 풀이:

**1단계 - initAcc 호출:**
- `accNum = 9981`, `bal = 2200.0`

**2단계 - xxx 호출 (amount = 100.0):**
- 조건 확인: `*en > 0` → `100 > 0` → 참
- 조건 확인: `*en < acc->bal` → `100 < 2200` → 참
- 두 조건 모두 참이므로 `if` 블록 실행
- `bal = 2200.0 - 100.0 = 2100.0`

**3단계 - yyy 호출:**
- `sim_pow(1.1, 3)` 계산:
  - i=0: r = 1.0 x 1.1 = 1.1
  - i=1: r = 1.1 x 1.1 = 1.21
  - i=2: r = 1.21 x 1.1 = **1.331**
- `bal = 2100.0 x 1.331 = 2795.10`

**4단계 - 출력:**
- `printf("%d and %.2f", 9981, 2795.10)` → **9981 and 2795.10**

## 16번 - Java 프로그래밍 (상속과 다형성)

```java
class classOne {
    int a, b;
    public classOne(int a, int b) { this.a = a; this.b = b; }
    public void print() { System.out.println(a*b); }
}
class classTwo extends classOne {
    int po = 3;
    public classTwo(int i) { super(i, i+1); }
    public void print() { System.out.println(po*po); }
}
public class main {
    public static void main(String[] args) {
        classOne one = new classTwo(10);
        one.print();
    }
}
```

### 답: `9`

### 풀이:
1. `new classTwo(10)` 실행:
   - `classTwo` 생성자 호출 → `super(10, 11)` → 부모 클래스의 `a=10`, `b=11` 설정
   - `po = 3` 초기화
2. `one.print()` 호출:
   - 변수 타입은 `classOne`이지만, 실제 객체는 `classTwo`
   - Java의 **동적 바인딩(Dynamic Binding)** 에 의해 `classTwo`의 `print()` 메서드가 호출됨
   - `po * po` = 3 x 3 = **9** 출력

> **핵심**: Java에서 오버라이딩된 메서드는 참조 변수의 타입이 아닌 **실제 객체의 타입**에 따라 호출된다 (런타임 다형성).

## 19번 - C언어 프로그래밍 (문자 변환)

```c
#include<stdio.h>
#include<ctype.h>
int main(){
    char*p = "It is 8";
    char result[100];
    int i;
    for(i=0; p[i]!='\0'; i++){
        if(isupper(p[i]))
            result[i] = (p[i]-'A'+5)%25 + 'A';
        else if(islower(p[i]))
            result[i] = (p[i]-'a'+10)%26 + 'a';
        else if(isdigit(p[i]))
            result[i] = (p[i]-'0'+3)%10 + '0';
        else
            result[i] = p[i];
    }
    result[i] = '\0';
    printf("%s\n",result);
    return 0;
}
```

### 답: `Nd sc 1`

### 풀이:
입력 문자열 `"It is 8"`을 한 글자씩 변환한다:

| 원문 | 종류 | 계산 | 결과 |
|------|------|------|------|
| I | 대문자 | ('I'-'A'+5)%25 + 'A' = (8+5)%25 + 'A' = 13 + 65 = 78 | **N** |
| t | 소문자 | ('t'-'a'+10)%26 + 'a' = (19+10)%26 + 'a' = 3 + 97 = 100 | **d** |
| (공백) | 기타 | 그대로 | **(공백)** |
| i | 소문자 | ('i'-'a'+10)%26 + 'a' = (8+10)%26 + 'a' = 18 + 97 = 115 | **s** |
| s | 소문자 | ('s'-'a'+10)%26 + 'a' = (18+10)%26 + 'a' = 2 + 97 = 99 | **c** |
| (공백) | 기타 | 그대로 | **(공백)** |
| 8 | 숫자 | ('8'-'0'+3)%10 + '0' = (8+3)%10 + '0' = 1 + 48 = 49 | **1** |

결과: **Nd sc 1**

> 이것은 시저 암호(Caesar Cipher)의 변형으로, 대문자/소문자/숫자 각각 다른 이동값(5, 10, 3)을 적용한다.

## 20번 - 소프트웨어 공학 (디자인 패턴)

### 답: `추상 팩토리 패턴 (Abstract Factory Pattern)`

### 풀이:
- **추상 팩토리 패턴**은 **생성(Creational) 패턴**의 하나로, 서로 관련되거나 의존적인 객체들의 조합을 만드는 인터페이스를 제공한다.
- **Kit 패턴**이라고도 불린다.
- 구체적인 클래스를 지정하지 않고 관련 객체들의 **제품군(family)**을 생성할 수 있다.

> **다른 생성 패턴과의 비교**:
> - **팩토리 메서드(Factory Method)**: 하나의 객체 생성을 서브클래스에 위임
> - **추상 팩토리(Abstract Factory)**: 관련된 **객체군** 생성의 인터페이스 제공
> - **빌더(Builder)**: 복잡한 객체의 생성 과정을 분리
> - **싱글톤(Singleton)**: 인스턴스가 하나만 존재하도록 보장 (1번 문제 참조)
