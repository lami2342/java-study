### 풀이

## 6번 - 네트워크 프로토콜

### 답: (1) ARP, (2) RARP

### 풀이
- **ARP (Address Resolution Protocol)**: IP 주소 → MAC 주소 변환
  - 네트워크 통신 시 목적지의 물리적 주소(MAC)를 알아내기 위해 사용
- **RARP (Reverse ARP)**: MAC 주소 → IP 주소 변환
  - 디스크가 없는 단말기가 자신의 MAC 주소로 IP 주소를 요청할 때 사용

## 13번 - Java 상속 및 오버라이딩

### 답: 54

### 풀이
1. `new Child()` 호출 → **Parent 생성자 먼저 실행**
2. Parent 생성자:
   - Parent의 `v = 1` → `++v` = 2 → `total += 2` → **total = 2**
   - `show()` 호출 → **동적 바인딩으로 Child의 show() 실행** (핵심!)
   - Child.show(): `total += total * 2` → `total += 2 * 2 = 4` → **total = 6**
3. Child 생성자:
   - Child의 `v = 10` (필드 초기화)
   - `v += 2` → v = 12
   - `total += v++` → `total += 12` → **total = 18**, v = 13
   - `show()` 호출 → Child.show(): `total += 18 * 2 = 36` → **total = 54**
4. 출력: **54**

> 핵심 포인트: Parent 생성자에서 `show()`를 호출하면, 실제 객체 타입(Child)의 오버라이딩된 메서드가 실행된다 (동적 바인딩).

## 20번 - Java 메서드 오버로딩 및 재귀

### 답: 4

### 풀이
핵심: `calc(String)`과 `calc(int)`는 **서로 다른 메서드**이다.

```
calc("5")  → String 버전 호출, value=5
  = calc(5-1) + calc(5-3)
  = calc(4) + calc(2)    ← 둘 다 int 버전!

calc(4): int 버전 (피보나치)
  = calc(3) + calc(2)
  = (calc(2) + calc(1)) + (calc(1) + calc(0))
  = ((calc(1) + calc(0)) + 1) + (1 + 0)
  = ((1 + 0) + 1) + 1
  = 3

calc(2): int 버전
  = calc(1) + calc(0) = 1 + 0 = 1

결과: calc("5") = 3 + 1 = 4
```

> 핵심 포인트:
> - `calc("5")`는 String 매개변수이므로 String 버전 호출
> - String 버전 내부의 `calc(value-1)`, `calc(value-3)`는 int 매개변수이므로 int 버전 호출
> - int 버전의 calc는 피보나치 수열 패턴 (F(n) = F(n-1) + F(n-2))
