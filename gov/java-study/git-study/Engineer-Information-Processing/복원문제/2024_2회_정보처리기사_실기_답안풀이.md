## 1번. Java 코드 출력값

### 답: `NNN`

### 풀이

```java
class Main {
    public static void main(String[] args) {
        int[] a = new int[]{1, 2, 3, 4};
        int[] b = new int[]{1, 2, 3, 4};
        int[] c = new int[]{1, 2, 3};
        check(a, b);
        check(a, c);
        check(b, c);
    }
    public static void check(int[] a, int[] b) {
        if (a==b) {
            System.out.print("O");
        }else{
            System.out.print("N");
        }
    }
}
```

- Java에서 `==` 연산자는 **참조(Reference) 비교**를 수행한다.
- `a`, `b`, `c`는 각각 `new`로 생성된 **서로 다른 객체**이므로, 배열의 내용이 같더라도 참조 주소가 다르다.
- `check(a, b)` → a와 b는 서로 다른 객체 → **N**
- `check(a, c)` → a와 c는 서로 다른 객체 → **N**
- `check(b, c)` → b와 c는 서로 다른 객체 → **N**
- 배열의 내용을 비교하려면 `Arrays.equals(a, b)`를 사용해야 한다.

> **출력 결과: NNN**

## 2번. 데이터베이스 용어

### 답: 반정규화 (Denormalization, 비정규화)

### 풀이

- **반정규화(Denormalization)**란 정규화된 데이터베이스에서 성능 향상을 위해 의도적으로 **데이터 중복을 허용**하는 기법이다.
- 주요 방법:
  - **테이블 병합**: 자주 조인하는 테이블을 하나로 합침
  - **테이블 분할**: 수직/수평 분할로 접근 성능 향상
  - **중복 컬럼 추가**: 자주 조회되는 컬럼을 다른 테이블에도 추가
  - **파생 컬럼 추가**: 계산 결과를 미리 저장
- **장점**: 조회 성능 향상, 조인 연산 감소
- **단점**: 데이터 무결성 저하, 저장 공간 증가, 갱신 비용 증가

## 3번. SQL 구문 빈칸 채우기

### 답: ① VALUES ② SELECT ③ FROM ④ SET

### 풀이

```sql
-- ① VALUES: INSERT 문에서 삽입할 값을 지정할 때 사용
INSERT INTO 사원 (사원번호, 이름, 주소, 부서) VALUES (32431, '정실기', '서울', '영업');

-- ② SELECT: 다른 테이블에서 데이터를 조회하여 삽입 (INSERT INTO ... SELECT 구문)
INSERT INTO 부서 (사원번호, 이름, 나이, 부서)
SELECT 사원번호, 이름, 나이, 23 FROM 사원 WHERE 이름 = '정실기';

-- ③ FROM: SELECT 문에서 조회할 테이블을 지정
SELECT * FROM 사원;

-- ④ SET: UPDATE 문에서 변경할 컬럼과 값을 지정
UPDATE 사원 SET 부서 = '퇴사' WHERE 사원번호 = 32431;
```

- `INSERT INTO ... VALUES (...)`: 직접 값을 지정하여 삽입
- `INSERT INTO ... SELECT ... FROM ...`: 서브쿼리 결과를 삽입
- `SELECT ... FROM 테이블명`: 테이블에서 데이터 조회
- `UPDATE 테이블명 SET 컬럼 = 값 WHERE 조건`: 데이터 수정

### 답: Cardinality: 5, Degree: 4

### 풀이

- **Degree(차수)**: 릴레이션의 **속성(Attribute/컬럼)의 수** → **4**
- **Cardinality(기수)**: 릴레이션의 **튜플(Tuple/행)의 수** → **5**

## 5번. 네트워크 보안

### 답: IPSec (IP Security)

### 풀이

- **IPSec**은 **네트워크 계층(Network Layer, 3계층)**에서 동작하는 보안 프로토콜이다.
- IP 패킷에 대한 **인증(Authentication)**과 **암호화(Encryption)**를 제공한다.
- 주요 프로토콜:
  - **AH(Authentication Header)**: 데이터 무결성과 인증을 제공 (암호화 X)
  - **ESP(Encapsulating Security Payload)**: 데이터 기밀성(암호화), 무결성, 인증을 모두 제공
- 동작 모드:
  - **전송 모드(Transport Mode)**: IP 페이로드만 보호
  - **터널 모드(Tunnel Mode)**: IP 패킷 전체를 보호 (VPN에서 주로 사용)

## 6번. Python 코드 출력값

### 답: `ab3ca3`

### 풀이

```python
def fnCalculation(x, y):
    result = 0;
    for i in range(len(x)):
        temp = x[i:i+len(y)]
        if temp == y:
            result += 1;
    return result

a = "abdcabcabca"
p1 = "ab";
p2 = "ca";
out = f"ab{fnCalculation(a,p1)}ca{fnCalculation(a,p2)}"
print(out)
```

**fnCalculation 함수 분석:**
- 문자열 `x`에서 길이 `len(y)`만큼의 부분 문자열을 하나씩 이동하며 추출하여, `y`와 일치하는 횟수를 센다.
- 즉, **패턴 매칭 횟수**를 반환하는 함수이다.

**fnCalculation(a, p1) - "ab" 찾기:**
- `a = "abdcabcabca"`
- i=0: `"ab"` == `"ab"` → 일치 (1)
- i=1: `"bd"` → 불일치
- i=2: `"dc"` → 불일치
- i=3: `"ca"` → 불일치
- i=4: `"ab"` == `"ab"` → 일치 (2)
- i=5: `"bc"` → 불일치
- i=6: `"ca"` → 불일치
- i=7: `"ab"` == `"ab"` → 일치 (3)
- i=8: `"bc"` → 불일치
- i=9: `"ca"` → 불일치
- i=10: `"a"` → 불일치
- **결과: 3**

**fnCalculation(a, p2) - "ca" 찾기:**
- i=0: `"ab"` → 불일치
- i=1: `"bd"` → 불일치
- i=2: `"dc"` → 불일치
- i=3: `"ca"` == `"ca"` → 일치 (1)
- i=4: `"ab"` → 불일치
- i=5: `"bc"` → 불일치
- i=6: `"ca"` == `"ca"` → 일치 (2)
- i=7: `"ab"` → 불일치
- i=8: `"bc"` → 불일치
- i=9: `"ca"` == `"ca"` → 일치 (3)
- i=10: `"a"` → 불일치
- **결과: 3**

**f-string 조합:**
- `f"ab{3}ca{3}"` → `"ab3ca3"`

## 7번. 암호화 알고리즘

### 답: AES (Advanced Encryption Standard)

### 풀이

- **AES**는 미국 NIST가 1997년 공모를 시작하여 2001년에 표준으로 선정한 **대칭키 블록 암호 알고리즘**이다.
- 벨기에 암호학자 Joan Daemen과 Vincent Rijmen이 개발한 **Rijndael** 알고리즘이 선정되었다.
- 주요 특징:
  - **블록 크기**: 128비트 고정
  - **키 길이**: 128비트, 192비트, 256비트 (가변)
  - **라운드 수**: 키 길이에 따라 10, 12, 14라운드
  - **DES의 후속** 표준 암호 알고리즘
- SPN(Substitution-Permutation Network) 구조를 사용한다.

## 8번. 패킷 교환 방식

### 답: ① 가상회선(Virtual Circuit) ② 데이터그램(Datagram)

### 풀이

**① 가상회선 방식 (Virtual Circuit)**
- 데이터 전송 전에 **논리적 경로(가상회선)**를 미리 설정한다.
- 모든 패킷이 **같은 경로**를 순서대로 따라간다.
- 패킷의 **순서가 보장**된다.
- 연결 설정 → 데이터 전송 → 연결 해제 과정을 거친다.
- 예: ATM, X.25, Frame Relay

**② 데이터그램 방식 (Datagram)**
- 사전 연결 설정 없이, 각 패킷이 **독립적으로 경로를 설정**하여 전달된다.
- 패킷마다 **다른 경로**로 전송될 수 있다.
- 패킷의 **순서가 보장되지 않는다** (수신 측에서 재조립 필요).
- 예: IP(인터넷 프로토콜), UDP

## 9번. 응집도

### 답: 순차적 응집도 (Sequential Cohesion)

### 풀이

- **순차적 응집도**란 모듈 내에서 한 활동의 **출력 데이터**가 다음 활동의 **입력 데이터**로 사용되는 경우를 말한다.
- 실행 순서가 밀접하게 관련되어 있다.

**응집도 순서 (높은 → 낮은):**

| 순서 | 응집도 | 설명 |
|------|--------|------|
| 1 | 기능적(Functional) | 모든 요소가 단일 기능 수행 |
| 2 | 순차적(Sequential) | 한 활동의 출력이 다음 활동의 입력 |
| 3 | 교환적(Communication) | 동일 데이터를 사용하는 활동들 |
| 4 | 절차적(Procedural) | 실행 순서가 정해진 활동들 |
| 5 | 시간적(Temporal) | 같은 시간에 실행되는 활동들 |
| 6 | 논리적(Logical) | 유사한 성격의 활동들을 하나로 모음 |
| 7 | 우연적(Coincidental) | 관련 없는 활동들이 모임 |

- 응집도는 **높을수록 좋다**.

## 11번. 라우팅 (RIP 최단 경로)

### 답: A → D → C → F

### 풀이

- **RIP(Routing Information Protocol)**은 **거리 벡터(Distance Vector)** 라우팅 프로토콜이다.
- **홉 수(Hop Count)**를 메트릭으로 사용하여 최단 경로를 결정한다.
- 최대 홉 수는 15이며, 16 이상은 도달 불가(Unreachable)로 간주한다.
- 주어진 네트워크 토폴로지에서 A → F까지 홉 수가 가장 적은 경로는 **A → D → C → F** (3홉)이다.

## 12번. SRT 스케줄링 평균 대기시간

### 답: 6.5

### 풀이

**SRT(Shortest Remaining Time) = 선점형 SJF**
- 새로운 프로세스가 도착할 때마다 남은 실행 시간이 가장 짧은 프로세스를 선택한다.

| 프로세스 | 도착 시간 | 서비스 시간 |
|---------|----------|------------|
| A       | 0        | 8          |
| B       | 1        | 4          |
| C       | 2        | 9          |
| D       | 3        | 5          |

**실행 과정:**

| 시간 | 실행 프로세스 | 설명 |
|------|-------------|------|
| 0~1 | A | A만 존재 (A 남은: 7) |
| 1~2 | B | B 도착(4), A 남은(7) → B 선택 (B 남은: 3) |
| 2~3 | B | C 도착(9), B 남은(3), A 남은(7) → B 계속 (B 남은: 2) |
| 3~5 | B | D 도착(5), B 남은(2) → B 계속, B 완료(시간 5) |
| 5~10 | D | D 남은(5), A 남은(7), C 남은(9) → D 선택, D 완료(시간 10) |
| 10~17 | A | A 남은(7), C 남은(9) → A 선택, A 완료(시간 17) |
| 17~26 | C | C 완료(시간 26) |

**대기시간 계산 (대기시간 = 완료시간 - 도착시간 - 서비스시간):**
- A: 17 - 0 - 8 = **9**
- B: 5 - 1 - 4 = **0**
- C: 26 - 2 - 9 = **15**
- D: 10 - 3 - 5 = **2**

**평균 대기시간** = (9 + 0 + 15 + 2) / 4 = 26 / 4 = **6.5**

## 13번. C언어 코드 출력값

### 답: 21

### 풀이

```c
#include <stdio.h>
int main() {
    int arr[3][3] = {1, 2, 3, 4, 5, 6, 7, 8, 9};
    int* parr[2] = {arr[1], arr[2]};
    printf("%d", parr[1][1] + *(parr[1]+2) + **parr);
    return 0;
}
```

**배열 구조:**
```
arr[0] = {1, 2, 3}
arr[1] = {4, 5, 6}
arr[2] = {7, 8, 9}
```

**포인터 배열:**
```
parr[0] = arr[1] → {4, 5, 6}을 가리킴
parr[1] = arr[2] → {7, 8, 9}을 가리킴
```

**각 항 계산:**
1. `parr[1][1]` = arr[2][1] = **8**
2. `*(parr[1]+2)` = *(arr[2]+2) = arr[2][2] = **9**
3. `**parr` = *parr[0] = *arr[1] = arr[1][0] = **4**

**결과:** 8 + 9 + 4 = **21**

## 16번. 결합도

### 답: 제어 결합도 (Control Coupling)

### 풀이

- **제어 결합도**란 한 모듈이 다른 모듈의 **논리적 흐름을 제어**하기 위해 **제어 요소(플래그, 스위치 등)**를 전달하는 결합도이다.
- 예: 함수 호출 시 `flag` 값을 전달하여 호출된 함수의 실행 흐름을 결정하는 경우

**결합도 순서 (낮은 → 높은, 낮을수록 좋음):**

| 순서 | 결합도 | 설명 |
|------|--------|------|
| 1 | 자료(Data) | 단순 데이터(매개변수)만 전달 |
| 2 | 스탬프(Stamp) | 자료구조(구조체 등)를 전달 |
| 3 | 제어(Control) | 제어 요소(플래그)를 전달 |
| 4 | 외부(External) | 외부에서 선언된 변수를 공유 |
| 5 | 공통(Common) | 전역 변수를 공유 |
| 6 | 내용(Content) | 다른 모듈 내부를 직접 참조/수정 |

- 결합도는 **낮을수록 좋다**.

## 20번. Java String split()

### 답: S

### 풀이

```java
String str = "ITISTESTSTRING";
String[] result = str.split("T");
System.out.print(result[3]);
```

**split("T") 동작:**
- 문자열 `"ITISTESTSTRING"`을 구분자 `"T"`로 분리한다.

```
I | T | IS | T | ES | T | S | T | RING
```

분리 결과:
```
result[0] = "I"
result[1] = "IS"
result[2] = "ES"
result[3] = "S"
result[4] = "RING"
```

- `"T"` 기준으로 나누면:
  - 첫 번째 `T` 앞: `"I"`
  - 첫 번째 `T`와 두 번째 `T` 사이: `"IS"`
